# 📋 콘텐츠 발견 시스템 근본 개선 기획서

**작성일**: 2026-03-29
**문제**: 100개 콘텐츠가 있지만 사용자가 접근할 메뉴/링크/버튼이 없음
**목표**: 글 추가 시 자동으로 웹사이트에 표시되는 시스템 구축

---

## 🔴 **현재 문제 상황**

### 문제 1: 콘텐츠와 UI가 분리됨
```
📁 public/content/           ← 100개 글이 여기 있음
   ├── doctor-ai/
   ├── ai-news/
   └── ...

❌ 하지만 사용자가 접근할 수 있는
   메뉴/링크/버튼이 없음!
```

### 문제 2: 매번 수동으로 링크를 추가해야 함
```
1. 글을 쓴다
   ↓
2. 오류 리포트를 쓴다 ("링크가 없어요!")
   ↓
3. HTML 파일에 수동으로 링크를 추가
   ↓
4. 다시 커밋
```

### 문제 3: 확장성이 없음
```
100개 글 → 1000개 글 → 1만 개 글
손으로 다 관리할 수 없음!
```

### 근본 원인

| 항목 | 현재 상태 |
|------|---------|
| **아키텍처** | 정적 HTML (수동 관리) |
| **메뉴 시스템** | 없음 |
| **검색 기능** | 없음 |
| **카테고리 페이지** | 없음 |
| **태그 필터** | 없음 |
| **자동화** | 0% |

---

## 🎯 **목표**

### 최종 상태
```
글을 쓰면
   ↓
자동으로 웹사이트에 표시됨
   ↓
메뉴, 검색, 필터에 모두 노출
```

### 핵심 요구사항
1. **자동화**: 글 추가 시 자동으로 메뉴에 나타남
2. **다양한 접근법**: 메뉴, 검색, 카테고리, 태그
3. **빠른 속도**: 로딩 시간 < 1초
4. **확장성**: 1만 개 글까지 처리 가능

---

## 📊 **해결 방안 3가지 비교**

### **Option 1: 정적 생성 (SSG)**

```
공정:
글 추가 → 빌드 실행 → 자동으로 HTML 생성

장점:
✅ 매우 빠름 (로딩 시간 0)
✅ 검색 엔진 최적화 (SEO)
✅ 서버 부하 없음
✅ 보안 강함

단점:
❌ 배포 필요 (매번 빌드 후 배포)
❌ 실시간 업데이트 안 됨
❌ 대규모 콘텐츠는 빌드 시간 오래 걸림

기술:
- Node.js 스크립트로 마크다운 → HTML 생성
- 메뉴 자동 생성
- 검색 인덱스 사전 생성

구현 난이도: ⭐⭐ (낮음)
```

### **Option 2: 동적 로딩 (Client-side)**

```
공정:
글 추가 → index.json 업데이트 → 브라우저에서 자동 로드

장점:
✅ 배포 불필요
✅ 실시간 업데이트
✅ 간편한 추가 (파일만 추가)
✅ 작은 파일 크기

단점:
❌ JavaScript 필요 (SEO 약함)
❌ 첫 로딩 시간 조금 있음
❌ 검색 기능은 클라이언트에서 처리 (무거움)
❌ 큰 데이터셋에서 느림

기술:
- Fuse.js (클라이언트 검색)
- JavaScript로 동적 메뉴 생성
- index.json 기반 필터링

구현 난이도: ⭐⭐⭐ (중간)
```

### **Option 3: 서버사이드 렌더링 (SSR)**

```
공정:
글 추가 → 서버에 배포 → 사용자 요청 시 동적 렌더링

장점:
✅ 실시간 업데이트
✅ SEO 최적화
✅ 강력한 검색 기능
✅ 동적 콘텐츠 지원

단점:
❌ 서버 비용 증가
❌ 복잡한 구현
❌ 캐싱 필요
❌ 배포 과정 복잡

기술:
- Next.js 또는 Express.js
- 데이터베이스 (또는 파일 시스템)
- 서버사이드 검색

구현 난이도: ⭐⭐⭐⭐⭐ (매우 높음)
```

---

## 💡 **추천: Option 1 (정적 생성) + Option 2 (동적 로딩)**

### 하이브리드 접근법

```
1단계: 정적 생성 (우선순위 높은 페이지)
└─ 홈페이지, 카테고리 페이지, 인기글 페이지
   → SSG로 빌드 (SEO 최적화)

2단계: 동적 로딩 (상세 페이지)
└─ 개별 글, 검색, 필터
   → JavaScript로 index.json에서 로드

3단계: 검색 기능 (클라이언트)
└─ Fuse.js로 실시간 검색
```

**이점**:
- ✅ 메인 페이지는 빠르고 SEO 좋음
- ✅ 글 추가할 때마다 배포 불필요
- ✅ 검색과 필터는 빠르고 유연함
- ✅ 구현 난이도 낮음

    ├── footer.html              (공통 푸터)
    └── styles.css

#### 필요한 파일
```

```

### **2~3일차: 정적 페이지 생성**

#### 1) 카테고리 페이지 생성
```javascript
// build-static-pages.cjs

각 카테고리별로:
- 제목
- 글 목록 (카드 형식)
- 필터링 옵션
- 검색 박스

생성 예시:
doctor-ai.html
  ├─ Fundamentals (10개)
  ├─ Prompts (5개)
  ├─ Cases (5개)
  └─ Tools (5개)
```

#### 2) 메인 페이지 업데이트
```html
<nav>
  <a href="/category/doctor-ai.html">Doctor AI</a>
  <a href="/category/ai-news.html">뉴스</a>
  <a href="/category/ai-tools.html">도구</a>
  <a href="/category/faq.html">FAQ</a>
  <a href="/category/tips.html">팁</a>
  <a href="/category/regulations.html">규제</a>
</nav>

<section class="featured">
  <!-- 최신 글 5개 자동 표시 -->
</section>
```

#### 3) 검색 페이지 생성
```html
<input type="search" placeholder="글을 검색하세요...">
<div id="results"></div>

<!-- 검색 결과 실시간 표시 -->
```

### **4~5일차: 동적 기능 추가**

#### 1) 클라이언트 검색 (Fuse.js)
```javascript
// search.js
const fuse = new Fuse(articles, {
  keys: ['title', 'description', 'tags']
});

input.addEventListener('input', (e) => {
  const results = fuse.search(e.target.value);
  displayResults(results);
});
```

#### 2) 필터링 기능
```javascript
// 카테고리별 필터
filterByCategory(category);

// 난이도별 필터
filterByDifficulty(difficulty);

// 읽기 시간별 필터
filterByReadTime(minutes);

// 태그별 필터
filterByTag(tag);
```

#### 3) 글 상세 페이지
```html
<!-- /article/[slug].html -->
동적으로 마크다운 로드
  ↓
HTML로 렌더링
  ↓
메타데이터 표시 (제목, 저자, 날짜, 읽기시간)
  ↓
네비게이션 (이전/다음 글)
```

## 핵심 원칙

### ⚡ 애린이 글의 5가지 원칙

#### 1️⃣ 한 글, 한 주제
```
❌ "LLM, Transformer, 그리고 RAG 완전 이해하기"
✅ "LLM이 뭐예요?"

❌ "의료 AI의 정규화와 윤리와 미래"
✅ "의료 AI 환각(Hallucination)이 뭐예요?"
```

**왜?**
- 뇌는 한번에 5-7개 정보만 처리 가능 (인지 부하 이론)
- 주제가 섞이면 초보자는 헷갈림
- "이 글이 뭘 말하는 거야?"라는 의문 발생

---

#### 2️⃣ 한 문장, 15단어 이내
```
❌ "머신러닝이라는 컴퓨터 과학의 중요한 분야는
    데이터로부터 자동으로 학습하는 알고리즘과
    통계 모델들의 집합으로 정의되며,
    이는 인공지능의 한 부분입니다."

✅ "머신러닝은 데이터로부터 자동 학습하는 AI입니다."

✅ "데이터를 보고 스스로 판단하는 컴퓨터."
```

**구조**: (주어) + (쉬운 동사) + (목적어)
- 주어: 무엇을
- 동사: 어떻게 (쉬운 표현)
- 목적어: 무엇이 되었나

---

#### 3️⃣ 이해하기 쉬운 비유/예시
```
❌ 기술적 설명
"Embedding은 고차원 벡터 공간에서 단어를
저차원으로 변환하는 과정입니다."

✅ 일상 비유
"Embedding은 복잡한 책을 한 페이지 요약본으로
만드는 것과 같습니다.

원래는 수백 페이지였지만,
가장 중요한 내용만 한 페이지에 담은 거죠."

✅ 의료 비유
"EHR(의료기록)을 AI가 한눈에 이해하도록
요약하는 과정이 Embedding입니다."
```

**비유 3단계**:
1. 초보자가 알고 있는 것 (책, 요약)
2. 그것과의 유사점 (내용 → 숫자)
3. 의료 적용 (EHR → 숫자 변환)

---

#### 4️⃣ 시각화 > 문장
```
❌ "AI 학습 과정은 여러 단계를 거치며,
    데이터 수집에서 시작하여 전처리,
    모델 학습, 평가, 배포 단계를 거칩니다."

✅ 다이어그램
┌─────────────┐
│  데이터수집  │
└──────┬──────┘
       ↓
┌─────────────┐
│  데이터정리  │
└──────┬──────┘
       ↓
┌─────────────┐
│  AI 학습   │
└──────┬──────┘
       ↓
┌─────────────┐
│  성능 확인   │
└──────┬──────┘
       ↓
┌─────────────┐
│  실제 사용   │
└─────────────┘
```

**시각화 종류**:
- 플로우차트 (단계, 흐름)
- 표 (비교, 특징)
- 다이어그램 (관계)
- 스크린샷 (실제 모습)

---

#### 5️⃣ 의료에서 쓰이는 형태로
```
❌ "ChatGPT는 자연어 처리 기술을 사용하여..."

✅ "ChatGPT는 의사의 진료기록 작성을 도와줄 수 있습니다.
   예를 들어:

   의사: '환자가 고혈압 약을 놓쳤대'
   ChatGPT: '환자는 항고혈압제 복용 빈번하게 누락됨'

   이렇게 약식 표현을 정식 의료기록으로 바꿔줍니다."
```

## 글의 구조

### 📐 기본 구조 (모든 글 필수)

```
1. 메타데이터 (Frontmatter)
2. 한줄 요약 (Hook)
3. 이 글을 읽어야 하는 이유 (Why)
4. 핵심 정의 (What)
5. 쉬운 예시 (Example)
6. 의료 활용 (Medical Application)
7. 주의할 점 (Caution)
8. 다음 단계 (Next Steps)
9. 참고자료 (References)
```

## 각 섹션별 작성법

### 1️⃣ 메타데이터 (Frontmatter)

```yaml
---
title: "LLM이 뭐예요?"
date: 2026-03-31
category: ai-terminology
subcategory: 기초개념
difficulty: beginner
tags: [ai, llm, beginner-friendly, concept]
reading-time: 3
description: "많은 데이터로 배워서 사람처럼 말하는 AI의 정체"
thumbnail: ""
draft: false
---
```

**각 필드 설명**:
- `title`: 질문형 (뭐예요?, 뭐가 다르죠?)
- `difficulty`: beginner (항상)
- `tags`: beginner-friendly 필수
- `reading-time`: 분 단위 (3, 5, 7)
- `description`: 한줄 요약 (한 문장, 15단어)

---

### 2️⃣ 한줄 요약 (Hook)

**목표**: 이 글을 읽어야 하는 이유를 15단어로

```markdown
## 한줄 요약

LLM은 엄청 많은 책을 읽고 배운 AI로,
당신처럼 질문에 답하고 글을 써줍니다.
```

**공식**:
```
[개념] + [비유/쉬운 설명] + [할 수 있는 것]
```

**예시**:
```
✅ "Token은 AI가 읽는 가장 작은 단위로,
   한 단어나 기호를 쪼갠 조각입니다."

✅ "Hallucination은 AI가 사실처럼 거짓말하는 것으로,
   자신 있게 틀린 답을 제시합니다."

✅ "RAG는 외부 자료를 찾아 AI의 답변을 개선하는 방법으로,
   병원 프로토콜을 AI에게 알려주는 식입니다."
```

### 3️⃣ "왜 알아야 할까?" 섹션

**목표**: 초보자가 "이 글 읽을 가치가 있나?"라고 느끼게 하기

**길이**: 2-3 문단 (150단어)

**공식**:
```
1. 당신의 상황 (Pain Point)
2. 이 개념이 관련된 이유
3. 알면 얻는 이점
```

**예시**:

```markdown
## 왜 알아야 할까?

🎯 당신이 ChatGPT를 쓰려고 하면,
"내 질문이 너무 길면 답변이 이상해진다"라고 생각해본 적 있나요?

💡 그건 바로 Token 때문입니다.

AI는 문장을 읽을 때, 전체 문장이 아니라
작은 조각들 (Token)로 쪼개서 읽어요.

마치 당신이 긴 의료기록을 읽을 때
"환자, 고혈압, 약" 이런 식으로 핵심 단어만 빠르게 읽듯이요.

✅ Token을 이해하면:
- ChatGPT 사용료를 절감할 수 있어요 (Token 개수 = 가격)
- 더 좋은 답변을 받을 수 있어요
- AI가 왜 답변을 못 했는지 이해할 수 있어요
```

### 4️⃣ "이게 뭐예요?" 섹션 (정의)

**목표**: 명확한 정의 + 초보자 친화적 비유

**길이**: 150-200단어

**구조**:
```
1. 한국말로 쉽게
2. 용어 설명 (괄호)
3. 비유 2-3개
4. 한 문장 정리
```

**예시**:

```markdown
## 이게 뭐예요?

### 정의
**LLM(거대언어모델)은 엄청 많은 글을 읽고 배운 AI입니다.**

AI에게 인터넷의 책, 논문, 뉴스 수조 개를 읽게 하고
"이런 패턴이 있더라" 라고 학습시킨 거예요.

### 비유 1: 책 읽은 학생
학생이 한국 전체 도서관의 책을 다 읽었다고 상상해보세요.
그럼 어떤 질문에도 도서관 지식으로 대답할 수 있겠죠?

LLM도 그래요.
인터넷의 거의 모든 글을 읽어서 질문에 답합니다.

### 비유 2: 의료기록 패턴
의사인 당신이 환자 1만 명의 의료기록을 읽었다고 하세요.
그럼 새 환자가 "고열, 기침" 이라고 하면
자동으로 "폐렴일 가능성 높다"고 생각이 드실 거예요.

LLM은 그 수준을 인터넷 규모로 한 거고,
의료 분야뿐 아니라 모든 분야에서 그래요.

### 한 줄 정리
💡 LLM = 책을 엄청 많이 읽은 AI = 뭐든 물어볼 수 있다
```

### 5️⃣ "예시를 들어볼게" 섹션

**목표**: 추상적 개념을 구체적으로 만들기

**길이**: 200-250단어

**구성**: 3-5개 다양한 예시 (복잡도 순서)

```markdown
## 예시를 들어볼게

### 예시 1: 일상에서
당신이 식당에서 메뉴 판독증이 있어서
휴대폰으로 사진을 찍으면
ChatGPT가 "이건 제육볶음이고,
이건 된장국 같네요"라고 말해줍니다.

이것도 다 LLM이 한 일이에요.

### 예시 2: 의료에서
의사: "최근 의료기록 정리해줄래?"
ChatGPT: "네, 이 환자는 고혈압,
당뇨가 있고 이전에 수술 경력이..."

→ LLM이 100페이지 기록에서
중요한 부분만 요약한 거예요.

### 예시 3: 더 심화된 사용
의사: "이 환자 케이스,
비슷한 사례 찾아줄래?"

LLM: "당신의 환자는 45세 여성,
고혈압 + 당뇨 조합입니다.
제가 알기로는 이런 환자들은...
(관련 의학 지식 서술)"

→ 학술 수준의 대화도 가능해요.
```

### 6️⃣ "의료에서는 이렇게 써요" 섹션

**목표**: 추상적 개념 → 병원 현장 적용

**길이**: 200-250단어

**중요**: 이것이 가장 중요한 섹션 (초보자가 "나도 쓸 수 있겠다"고 느끼는 부분)

```markdown
## 의료에서는 이렇게 써요

### 활용 1: 진료기록 작성
당신: 오늘 외래 환자 50명, 기록 이제 시작...😫

해결책: ChatGPT에게
"환자가 이렇게 말했어: 요즘 자주 피곤하고,
밤에 잠을 못 자. 최근에 스트레스 많음.
고혈압 약은 먹고 있음"

→ ChatGPT가 정식 의료기록 형태로 정리해줍니다:
"주호소: 피로감, 불면증
현병력: 스트레스 연관 증상으로 보여짐
과거력: 고혈압 (약물 복용 중)"

### 활용 2: 환자 교육
당신: "당뇨가 뭔지 쉽게 설명해줄래?"

LLM: "당뇨는 혈당 조절이 안 되는 병입니다.
마치 당신 집의 온도 조절이 안 되는 것처럼,
혈당도 높아졌다 낮아졌다 하는 거죠.
이 때문에..."

→ 환자도 이해할 수 있는 설명이 되어요.

### 활용 3: 문헌검색
당신: "환자가 희귀병인데,
최신 치료법이 뭘까?"

LLM: "2024년 기준으로는
다음과 같은 치료들이 있습니다..."
(참고자료 추천)

→ PubMed 검색보다 빨리 정보 수집 가능해요.
```

### 7️⃣ "주의할 점" 섹션

**목표**: 초보자가 하는 흔한 실수 방지

**길이**: 150-200단어

**구성**: 3-5개의 오해/실수

```markdown
## 주의할 점 ⚠️

### 오해 1: "LLM이 의사를 대체할까?"
❌ 절대 아닙니다.
LLM은 도구일 뿐, 의료 판단은 의사만 합니다.
AI의 답변이 틀릴 수도 있으니까요.

✅ 올바른 사용:
"LLM이 말한 것 + 의사의 판단" = 최고의 진료

### 오해 2: "환자 정보 입력해도 괜찮을까?"
❌ 위험합니다!
ChatGPT 같은 무료 AI에 개인정보를 입력하면
회사 서버에 저장될 수 있습니다.

✅ 올바른 사용:
"환자 이름 없이, 증상만" 입력하세요.
"55세 여성, 고혈압 약 복용,
요즘 어지러움..." (O)
"김○○ 환자, 주소는 서울..." (X)

### 오해 3: "LLM의 최신 정보는 항상 맞을까?"
❌ 아니요, 학습 시점 이후의 정보는 부정확할 수 있어요.
또한 "환각 (hallucination)"이라고
완전히 거짓을 사실처럼 말할 수도 있습니다.

✅ 올바른 사용:
항상 "이게 맞나?" 하고 의심하세요.
중요한 결정 전에는
다른 출처로 검증하세요.
```

### 8️⃣ "더 알고 싶다면" 섹션

**목표**: 다음 단계 학습 제시

## 한줄 요약

Token은 AI가 문장을 읽을 때 쪼갠 가장 작은 조각으로,
마치 당신이 의료기록을 단어별로 읽듯이
AI도 Token별로 읽고 이해합니다.

---

## 왜 알아야 할까?

🎯 ChatGPT를 5번 쓰면 비용이 드나요?

혹은 이런 경험 있으신가요:
"의료기록을 길게 입력했더니
ChatGPT가 갑자기 이상한 답을 해줬어."

💡 이 모든 것이 바로 **Token** 때문입니다.

Token을 이해하면:
- ✅ ChatGPT 사용료를 50% 절감할 수 있어요
- ✅ 더 정확한 답변을 받을 수 있어요
- ✅ AI가 왜 답변을 못했는지 알 수 있어요
- ✅ 프롬프트를 더 효율적으로 쓸 수 있어요

---

## 이게 뭐예요?

### 정의
**Token은 AI가 읽는 가장 작은 단위입니다.**

한국어로는 "토큰" 또는 "조각"이라고 부르는데,
AI는 당신의 문장을
"조각 + 조각 + 조각..."으로 쪼개서 읽어요.

### 비유 1: 의료기록 읽기
당신이 의료기록을 빠르게 읽을 때를 생각해보세요:

전체 문장: "환자는 최근 2주간 지속적인 두통으로 내원하였습니다."

당신의 뇌가 처리하는 방식:
"환자" / "2주간" / "두통" / "내원"
→ 이 4-5개 단어가 의미의 조각들이에요.

AI의 Token도 비슷해요:
"환자" / "는" / "최근" / "2" / "주간" / ...

마치 당신이 핵심 단어들에만 주목하듯이,
AI는 Token들에만 집중합니다.

### 비유 2: 문자 메시지
카톡에서 "ㅋㅋㅋㅋ"라고 쓰면,
서버는 이걸 여러 글자 조각으로 나눠요:
"ㅋ" / "ㅋ" / "ㅋ" / "ㅋ"

AI도 그래요. 당신의 입력을 조각내서 이해합니다.

### 한 줄 정리
💡 Token = AI가 이해하기 위해 쪼갠 가장 작은 조각

---

## 예시를 들어볼게

### 예시 1: 간단한 문장
문장: "고혈압입니다"

→ Token으로 나뉨:
"고" / "혈압" / "입니다" (3개)

또는

"고혈" / "압" / "입니다" (3개)

(정확한 나뉨은 AI 모델에 따라 다릅니다)

### 예시 2: 의료기록
의료기록 원문:
"55세 남성, 2주간의 흉부 통증으로 입원.
BP 140/90, HR 88. 심전도 정상."

→ Token 개수 (대략): 35개

이 경우 ChatGPT 사용료는:
입력 35 Token × 가격 = 약 1원 정도

### 예시 3: 긴 의료기록
처방전, 검사 결과, 진료 기록, 수술 기록 등
전부를 ChatGPT에 입력하면?

→ Token 개수: 500-2,000개

사용료: 500-2,000원 정도

즉, 한 번의 consultation이 몇천 원이 될 수 있어요!

---

## 의료에서는 이렇게 써요

### 활용 1: 사용료 절감
❌ 비효율적 (많은 Token 사용):
"과거 병력, 현병력, 가족력, 약물 복용력,
알레르기 여부, 근래 검사 결과,
수술 병력, 입원 병력을 모두 입력"
→ 1,000 Token 사용

✅ 효율적 (적은 Token 사용):
"주증상: 두통, 2주간 지속,
고혈압 약 복용 중, 최근 스트레스 증가"
→ 25 Token 사용

**40배 차이!**

### 활용 2: 정확한 답변 받기
❌ 너무 많은 정보 → AI가 "혼란"
"의료기록 전체 + 여러 병력 + 약물 목록"
→ AI가 핵심을 못 잡을 수 있어요

✅ 핵심만 입력 → AI가 집중
"주증상: ○○
현병력: ○○
과거력: ○○
질문: ○○"
→ 더 정확한 답변

### 활용 3: 토큰 한계 이해
ChatGPT의 한계:
- 한 번에 처리 가능한 Token: 약 4,000개
- 의료기록 전체는 보통 3,000-5,000 Token

따라서:
"의료기록 전체 요약하기" (O)
"의료기록 전체 + 다른 3개 환자 기록" (X - 초과됨)

---

## 주의할 점 ⚠️

### 주의 1: "더 많은 정보 = 더 좋은 답"은 아닙니다
❌ 환상:
"정보가 많을수록 AI가 더 정확하게 답할 것"

✅ 현실:
초과 정보는 AI를 오히려 혼란스럽게 만들어요.
(cognitive overload라는 현상)

마치 당신이 한번에 너무 많은 정보를 받으면
정신이 없어지듯이요.

### 주의 2: "Token 개수 = 단어 개수"는 아닙니다
한국어: "안녕하세요" = 2-3 Token
(언어마다, 모델마다 다릅니다)

### 주의 3: Token 한계를 고려하세요
매우 긴 환자 병력이 있다면:
1. 요약본을 먼저 작성
2. 그 다음에 ChatGPT에 입력
→ Token 절감 + 더 좋은 답변

## 실전 예제

### ✏️ 실제 쓰인 글 분석

글: "Token이 뭐예요?" (위의 예제 참고)

**분석**:

#### 구조 분석
```
한줄 요약: 1문단 (명확함)
왜 알아야 할까: 3문단 (pain point 명확)
정의: 4개 섹션 (다면 설명)
예시: 3개 (단순 → 의료 → 복잡)
의료 활용: 3개 (실전)
주의점: 3개 (오해 해소)
다음: 글추천 (연결)

총 길이: ~1,600단어 (4-5분)
이모지: 적절하게 분산
```

#### 문장 길이 분석
```
"Token은 AI가 문장을 읽을 때 쪼갠 가장 작은 조각으로"
→ 16단어 (약간 길지만, "으로" 마지막이라 OK)

"마치 당신이 의료기록을 단어별로 읽듯이"
→ 11단어 (좋음)

"AI도 Token별로 읽고 이해합니다."
→ 8단어 (좋음)
```

#### 비유 개수
- 의료기록 읽기: 1개 (의료)
- 문자 메시지: 1개 (일상)
- 총 2개 ✅

#### 예시 개수
- 간단한 문장: 1개 ✅
- 의료기록: 1개 ✅
- 긴 의료기록: 1개 ✅

# A급 기술·의료 블로그 글 체크리스트

오케스트레이션 시리즈와 이후 글에 공통으로 쓸 **최소 품질 패키지**다. 전부 채우면 “교과서 요약”이 아니라 **한 편에 납득 가능한 주장 하나**가 남는다.

## 1. 서론 3문장 (`## 들어가며`)

1. **누가** 어떤 **상황**에서 막히는지 (독자 1인칭 또는 관찰 1문장).
2. 이 글이 **다루는 범위**와 **다루지 않는 것** (한 문장).
3. 읽고 나면 **무엇을 결정·설계할 수 있는지** (한 문장).

## 2. 근거 1개 (`### 근거 한 줄`)

- 논문·공식 문서·표준(HL7 FHIR 등)·책 **제목 + 연도(또는 arXiv 번호)** 를 명시한다.
- 숫자(%, 배)를 쓸 때는 **어느 벤치마크·조건**인지 붙인다. 임상과 동일 조건이 아니면 그렇게 쓴다.

## 3. 반례 1개 (`### 반례 — 이 패턴이 틀리는 경우`)

- “이 구조를 쓰면 안 되는” **구체 시나리오** 한 가지 (비용, 디버깅, 규제, 단순 과제 등).

## 4. 실무 체크리스트 (`### 실무 체크리스트`)

- 4~7개, **행동형** (예: “실패 시 재시도 상한을 정했는가?”).
- 복붙해 회의·설계 노트에 쓸 수 있게 짧게.

## 5. 태그 `cu` + 편집 시각 (Cursor 초안 추적)

편집이 들어갔는지 확인하려고 **같은 시각 번호**를 세 곳에 맞춘다.

- **형식:** `cu` + `YYMMDDHHmm` (2자리 연·월·일·시(24h)·분). 예: `2604011702` → 태그 키는 `cu2604011702` (26년 4월 1일 17:02).
- `tags` 배열 **맨 앞**에 `"cu2604011702"` 처럼 **전체 문자열 하나**로 둔다.
- `key_takeaways`에 한 줄: `태그 cu2604011702: Cursor 초안 — …` (해당 글 주의 한 줄).
- 본문 말미: `*편집 초안(cu2604011702). …*`

**일괄 반영:** 저장소 루트 `blog/`에서 `node scripts/apply-cu-timestamp.mjs` (실행 시점의 로컬 시각이 붙는다). 고정 시각이 필요하면 PowerShell에서 `$env:CU_TS='2604011702'; node scripts/apply-cu-timestamp.mjs`.

---

시리즈 허브 글(`what-is-orchestration`)은 위 항목을 **전부** 채운 템플릿으로 유지하고, 나머지 편은 최소한 **cu + 시각 + key_takeaways + 푸터**, 가능하면 **들어가며·근거·반례·체크리스트**를 순차로 보강한다.

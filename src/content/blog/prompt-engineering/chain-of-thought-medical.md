## 한줄 요약
"단계별로 생각해봐" 한 문장이 AI의 진단 추론 품질을 크게 바꾼다 — Chain-of-Thought는 AI가 결론으로 점프하지 않고 추론 과정을 보여주게 만드는 기법이다.

## 본문

### CoT가 필요한 이유

AI는 기본 설정에서 빠른 답을 선호한다. "이 환자의 진단은?" 이라고 물으면 가장 흔한 진단을 바로 뱉는다. 하지만 임상은 그렇지 않다.

좋은 임상 추론은 Step-by-step 과정이다:
1. 주소와 기간 파악
2. 동반증상으로 감별진단 생성
3. 검사 결과로 좁히기
4. 최종 진단 및 근거 제시

CoT 프롬프팅은 AI에게도 이 과정을 명시적으로 밟게 만든다.

---

### 기본 CoT vs 일반 프롬프트 비교

케이스: 55세 여성, 갑작스러운 흉통 30분, 발한, 좌측 팔 저림. BP 160/100.

일반 프롬프트
```
이 환자의 진단은 무엇인가요?
```
→ AI 답변: "급성 심근경색 가능성이 높습니다. 심전도와 트로포닌 검사가 필요합니다."

빠르지만 과정이 없다. 왜 그 진단인지, 다른 가능성은 왜 배제됐는지 모른다.

CoT 프롬프팅
```
다음 환자를 단계별로 추론해 주세요.
1단계: 주요 증상과 특징 정리
2단계: 주요 감별진단 3~5개와 각 근거
3단계: 각 진단에 필요한 검사
4단계: 가장 유력한 진단과 이유

환자: 55세 여성, 갑작스러운 흉통 30분, 발한, 좌측 팔 저림. BP 160/100.
```

→ AI가 각 단계를 밟으며 추론 과정을 보여준다. AMI 외에 대동맥박리, 폐색전증도 같이 다루고 감별 포인트를 설명한다.

---

### "단계별로 생각해봐" 트리거 문장들

CoT를 유도하는 효과적인 트리거:

| 트리거 문장 | 효과 |
|-------------|------|
| `"단계별로 생각해봐"` | 기본 CoT 활성화 |
| `"추론 과정을 보여줘"` | 중간 단계 명시 |
| `"왜 그 진단인지 설명해줘"` | 근거 제시 강제 |
| `"다른 가능성도 먼저 나열해봐"` | 감별진단 포함 |
| `"결론 전에 pros/cons를 따져봐"` | 양방향 검토 |

가장 간단하고 강력한 방법: 프롬프트 끝에 "단계별로 생각해봐." 한 문장 추가.

---

### 의료 특화 CoT 템플릿

감별진단 CoT
```
당신은 내과 전문의입니다. 다음 환자를 아래 단계로 추론해 주세요.

1단계 — 핵심 증상 요약 (2~3줄)
2단계 — 감별진단 (가능성 높은 순 4~5개, 각 진단의 근거 1줄)
3단계 — 배제 검사 (각 진단별 필요 검사)
4단계 — 최우선 진단과 즉각적 처치

환자: [환자 정보]
단계별로 생각해봐.
```

약물 선택 CoT
```
다음 환자에게 가장 적절한 항생제를 선택해 주세요. 단계별로:
1단계: 감염 부위와 흔한 원인균
2단계: 항생제 옵션들과 각각의 장단점
3단계: 이 환자의 특수 상황(알레르기/기저질환) 고려
4단계: 최종 선택과 용량/기간

환자: [정보]
```

치료 방침 결정 CoT
```
아래 임상 상황에서 치료 방침을 결정해야 합니다.
찬반 양쪽을 먼저 논한 후 결론을 내려주세요.

찬성 측 근거 → 반대 측 근거 → 최종 권고 → 모니터링 계획

상황: [케이스]
```

### 고급: Zero-shot CoT vs Few-shot CoT

Zero-shot CoT: 트리거 문장만 추가
```
...단계별로 생각해봐.
```

Few-shot CoT: 추론 과정 예시를 먼저 보여줌
```
[예시 케이스]
환자: 30세 남성, 발열 38.8도, 기침 3일, 양측 폐 침윤.

1단계: 청년 남성, 발열+기침+폐 침윤 → 폐렴 가능성 높음
2단계: CAP (폐렴구균), 비정형 폐렴 (마이코플라스마), 결핵
3단계: 객담배양+흉부CT, 콜드응집소, 투베르쿨린
4단계: 비정형 폐렴 의증 → azithromycin 처방, 48시간 추적

[실제 케이스]
환자: [환자 정보]
같은 형식으로 추론해 주세요.
```

Few-shot CoT는 추론의 깊이와 형식 모두를 통제할 수 있어 교육용·컨퍼런스 준비에 유용하다.

---

### CoT의 한계

- 길어진다: 빠른 답이 필요할 땐 불필요한 오버헤드
- 환각도 단계별로: 잘못된 전제를 자신 있게 설명하기도 한다
- 검증 필수: 과정이 그럴듯해도 결론이 틀릴 수 있다. 특히 드문 질환, 최신 가이드라인 변경 사항은 반드시 확인

## 핵심 정리
- CoT = AI에게 추론 과정을 밟게 만드는 기법
- "단계별로 생각해봐" 한 문장이 가장 간단한 CoT 트리거
- 감별진단, 약물 선택, 치료 방침 결정처럼 다단계 추론이 필요한 상황에 특히 효과적
- 결과는 항상 검증해야 한다 — 과정이 그럴듯해도 결론이 틀릴 수 있다

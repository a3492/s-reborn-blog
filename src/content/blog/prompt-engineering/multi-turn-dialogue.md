## 한줄 요약
AI와의 대화는 한 번으로 끝내지 마라 — 첫 답변에 추가 정보를 주면서 좁혀가면, 컨퍼런스 토론처럼 깊은 임상 논의가 된다.

## 본문

### 싱글턴 vs 멀티턴

싱글턴 방식 — 한 번에 모든 걸 물어보는 방식
```
55세 남성, 흉통, 발한, 좌측 팔 저림, BP 160/100, ECG에서 ST 상승,
트로포닌 양성, 당뇨 있음. 진단과 치료 계획은?
```
→ 정보를 다 주니까 답변은 나오지만, 추론 과정이 없고 임상 사고가 개입할 여지가 없다.

멀티턴 방식 — 단계별로 정보를 추가하는 방식
```
[턴 1] 55세 남성, 갑작스러운 흉통 30분, 발한. 우선 무엇을 확인하겠는가?
[턴 2] BP 160/100, HR 92, SpO2 97%. 이 정보로 감별진단은?
[턴 3] ECG를 봤더니 V2-V5에서 ST 상승 3mm. 지금 가장 중요한 조치는?
[턴 4] 트로포닌 I 4.2 ng/mL (정상 0.04 미만). 진단 확정하고 치료 계획 수립해줘.
```
→ 단계별로 추론이 쌓이면서 실제 임상 사고 과정이 재현된다. 교육·컨퍼런스에 훨씬 유용하다.

---

### 멀티턴이 유용한 상황

| 상황 | 활용법 |
|------|--------|
| 복잡한 케이스 논의 | 정보를 단계별로 추가하며 감별진단 좁히기 |
| 레지던트 교육 | AI로 OSCe 시나리오 시뮬레이션 |
| 논문 리뷰 | 섹션별로 나눠 분석 후 종합 |
| 치료 계획 수립 | 초안 → 피드백 → 수정 반복 |
| 환자 교육 자료 작성 | 초안 → 어조 조정 → 길이 조정 |

---

### 컨텍스트 관리 팁

1. 앞 대화를 명시적으로 참조

AI는 대화 맥락을 유지하지만, 긴 대화에서는 앞 내용을 잊거나 혼동할 수 있다.
```
"앞서 말한 환자(55세 남성, STEMI)에 추가로..."
"첫 번째 감별진단(AMI)을 전제로 계속 논의하겠습니다."
```

2. 중간 정리 요청

긴 대화 후에는 요약을 요청해 컨텍스트를 정리한다.
```
"지금까지 논의한 내용을 3줄로 요약해줘.
그 위에서 다음 단계를 이어가자."
```

3. 새 화제 시작 시 명확하게 전환

```
"이전 케이스는 끝났습니다. 새로운 케이스 시작합니다."
```
명확한 전환 없이 새 케이스를 시작하면 AI가 앞 케이스와 혼동할 수 있다.

---

### 실전: 케이스 컨퍼런스 시뮬레이션

```
[설정]
당신은 내과 교수입니다. 나는 전공의이고, 케이스 컨퍼런스를 진행합니다.
내가 케이스 정보를 단계별로 드릴 테니, 매 턴마다:
1. 이 정보로 가능한 감별진단 2~3개
2. 다음으로 꼭 확인해야 할 것
을 말씀해 주세요. 설명은 교육적으로 해주세요.
```

→ 이 설정 하나로 여러 턴에 걸친 교육용 케이스 시뮬레이션이 가능하다.

---

### 치료 계획 반복 개선

멀티턴은 초안을 점진적으로 다듬는 데도 유용하다.

```
[턴 1] 이 환자의 고혈압 치료 계획 초안을 작성해줘.

[턴 2] 좋아. 이제 이 환자가 eGFR 35로 신기능이 저하됐다는 걸 반영해서 수정해줘.

[턴 3] 환자가 ARB에 알레르기가 있다고 한다. 대안을 포함해서 다시 작성해줘.

[턴 4] 최종 계획을 표 형식으로 정리해줘.
```

처음부터 모든 조건을 넣지 않고, 현실에서 정보가 추가되는 것처럼 대화를 진행하면 실제 진료 흐름과 유사해진다.

---

### 멀티턴의 한계

- 컨텍스트 윈도우 한계: 매우 긴 대화(수십 턴)는 앞 내용이 잘릴 수 있다
- 오류 누적: 초반 잘못된 전제가 이후 대화 전체에 영향을 준다
- 대화 재시작 문제: 새 세션 시작 시 이전 대화 맥락이 사라진다 — 중요 케이스는 요약본을 저장해두는 것이 좋다

## 핵심 정리
- 멀티턴 = 한 번에 모든 걸 묻지 않고, 단계별로 정보를 추가하며 대화를 이어가는 방식
- 케이스 컨퍼런스, 교육, 치료 계획 수립에 특히 효과적
- 긴 대화 후 중간 정리 요청으로 컨텍스트를 유지하는 것이 중요

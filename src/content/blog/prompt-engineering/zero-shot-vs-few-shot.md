## 한줄 요약
Zero-shot은 간단한 요청에, Few-shot은 AI가 원하는 형식·논리를 모를 때 쓴다 — 예시 2~3개가 긴 설명 10줄보다 낫다.

## 본문

### 개념 정리

**Zero-shot 프롬프팅**은 예시 없이 바로 요청하는 방식이다. AI 모델이 사전학습으로 이미 충분히 아는 내용이라면 예시 없이도 잘 동작한다.

**Few-shot 프롬프팅**은 원하는 입출력 예시를 1~5개 먼저 보여주고 나서 실제 요청을 넣는 방식이다. AI가 "아, 이런 형식으로 답하면 되는구나"를 패턴으로 학습한다.

---

### 의료 현장 비교 예시

**상황: SOAP 노트 작성**

**Zero-shot (예시 없이)**
```
다음 환자 정보를 SOAP 노트로 정리해 주세요.
환자: 45세 남성, 두통 3일째, BP 145/90, 진통제 복용 중
```
→ AI가 알아서 SOAP 형식을 쓰긴 하지만, 병원마다 선호하는 세부 포맷이 다를 수 있다.

**Few-shot (예시 포함)**
```
아래 형식으로 SOAP 노트를 작성해 주세요.

[예시 입력]
환자: 32세 여성, 복통 1일째, 오심 동반, 우하복부 압통

[예시 출력]
S: 복통 발생 1일 전부터 시작된 우하복부 통증, 오심 동반. VAS 6/10.
O: 우하복부 압통(+), 반발통(±). BT 37.8°C.
A: 급성 충수염 의증 (r/o 난소낭종)
P: CBC/CRP/복부초음파 시행, 외과 consult

[실제 입력]
환자: 45세 남성, 두통 3일째, BP 145/90, 진통제 복용 중
```
→ 내 병원 형식 그대로 출력된다.

---

### 언제 어떤 방식을 쓸까

| 상황 | 추천 방식 | 이유 |
|------|-----------|------|
| 감별진단 나열 | Zero-shot | AI가 이미 잘 안다 |
| 특정 형식의 노트 작성 | Few-shot | 내 병원 포맷을 알려줘야 한다 |
| 논문 요약 | Zero-shot | 표준 형식으로도 충분 |
| 커스텀 채점 기준 적용 | Few-shot | 기준을 예시로 보여야 한다 |
| 환자 설명문 작성 | Few-shot | 어투·난이도를 예시로 지정 |
| 약물 상호작용 확인 | Zero-shot | 사실 기반 조회라 형식 무관 |

---

### Few-shot 작성 팁

**1. 예시는 2~3개면 충분**
예시가 많다고 좋은 게 아니다. 5개 이상이면 컨텍스트를 낭비하고 오히려 헷갈릴 수 있다.

**2. 예시는 실제로 잘 된 결과물로**
나쁜 예시를 "이렇게 하면 안 된다"로 넣으면 역효과가 난다. 좋은 예시만 넣어라.

**3. 입력-출력 쌍을 명확하게 구분**
`[예시 입력]` / `[예시 출력]` 레이블을 붙이거나, `Q:` / `A:` 형식을 써서 AI가 혼동하지 않도록 한다.

**4. 실제 요청과 예시의 형식을 통일**
예시는 `이름/나이/증상` 순서인데, 실제 입력은 다른 순서면 성능이 떨어진다.

---

### 실전: 외래 진료기록 요약

```
아래 규칙으로 외래 진료를 3줄 요약해 주세요.
형식: 주소 → 주요 소견 → 처치/계획

[예시]
입력: 67세 여성. 무릎 통증 3달째. X-ray에서 내측 관절강 협소. NSAIDs 처방.
출력: 만성 무릎 통증 / 방사선 내측 골관절염 소견 / NSAIDs 처방, 4주 후 추적

[예시]
입력: 28세 남성. 발열 38.5도, 인후통 2일. 편도 발적. Strep 신속검사 양성.
출력: 급성 편도염 (Strep 양성) / 발열·인후통 / 항생제(Amoxicillin) 처방, 완료 확인

[실제 입력]
45세 남성. 두통 3일째. BP 145/90. 뒤통수 두통, 긴장형 양상. 진통제 복용 중에도 지속.
```

## 핵심 정리
- **Zero-shot**: 빠르고 간편, AI가 이미 잘 아는 작업에 충분
- **Few-shot**: 형식·어투·기준을 내 것으로 맞출 때. 예시 2~3개가 설명보다 효과적
- 시작은 Zero-shot → 원하는 결과가 안 나오면 Few-shot으로 업그레이드

## 한줄 요약
AI는 논문을 대신 써주는 도구가 아니라, 막힌 부분을 뚫어주는 도구다 — 주제 탐색부터 영어 교정까지 5단계를 AI와 함께 돌파하라.

## 본문

### 전공의에게 논문이란

레지던트 4년차, 졸업 논문 마감은 두 달 뒤다. 외래와 당직으로 하루가 지나가고, 책상 앞에 앉으면 빈 화면만 남아 있다. 무엇부터 시작해야 할지 모르고, PubMed를 열면 쏟아지는 영어 논문 더미에 막힌다. 결국 선배에게 "어떻게 하셨어요?"를 물어보지만 돌아오는 답은 "그냥 하다 보면 돼."

이 막막함은 능력의 문제가 아니다. 체계의 문제다.

AI는 이 체계를 제공한다. 아이디어를 다듬고, 문헌을 정리하고, 논리 구조를 잡고, 영어를 다듬는 작업 — 이 모든 단계에서 AI는 막힌 부분을 뚫는 실질적 도구가 된다. 단, AI가 논문을 써주는 것이 아니다. 당신이 쓰고, AI가 돕는다.

---

### 단계 1 — 주제 탐색: 아이디어를 임상 질문으로 바꾸기

좋은 논문은 좋은 질문에서 시작한다. 임상에서 마주친 의문 — "왜 이 환자는 표준 치료에 반응이 없지?", "우리 병원 환자들은 가이드라인과 달리 이런 패턴을 보이는데?" — 이런 관찰이 연구 주제가 된다.

AI는 막연한 관찰을 PICO 형식의 연구 질문으로 구체화하는 데 탁월하다.

프롬프트 — 주제 구체화
```
나는 가정의학과 전공의입니다. 외래에서 복용 중인 스타틴 용량을 실제로 가이드라인 권고대로 titration하지 않는 경우가 많다고 느끼고 있습니다.

이를 바탕으로 연구 가능한 PICO 질문 3가지를 만들어 주세요.
각각에 대해:
- 연구 가능성 (데이터 접근성, 표본 크기)
- 예상 연구 설계 (후향적 코호트, 단면 연구 등)
- 비슷한 선행 연구가 있을 가능성
을 간략히 평가해 주세요.
```

프롬프트 — 연구 공백(Research Gap) 탐색
```
아래 주제에 대한 연구 공백을 찾는 데 도움을 주세요:
주제: [주제 입력]

다음 관점에서 분석해 주세요:
1. 기존 연구에서 자주 다루어진 측면
2. 상대적으로 연구가 부족한 하위 집단이나 상황
3. 방법론적 한계가 자주 언급되는 부분
4. 한국 또는 아시아 환자 집단 특이성이 논의된 경우

각 항목을 2~3문장으로 설명해 주세요.
```

이 단계에서 AI의 답변은 시작점이다. AI가 언급한 논문 제목은 반드시 PubMed에서 직접 검색해 실재 여부를 확인해야 한다.

---

### 단계 2 — 문헌 검토: Literature Review를 체계적으로

PubMed에서 수십 편의 논문을 찾았다. 이제 어떻게 읽고 정리하느냐가 관건이다. AI는 두 가지 방식으로 돕는다.

방법 A — Abstract 구조화 요약

논문 Abstract를 붙여넣으면 AI가 핵심만 정리해준다.

프롬프트 — 논문 요약
```
아래 논문 Abstract를 읽고 다음 형식으로 정리해 주세요:

1. 연구 목적 (1문장)
2. 연구 설계 (연구 유형, 대상 환자 수, 추적 기간)
3. 주요 결과 (수치와 통계적 유의성 포함)
4. 저자들의 결론 (1~2문장)
5. 이 연구의 한계점 (Abstract에서 언급된 것 + 내가 추가로 생각해볼 것)
6. 내 연구와의 관련성 (내 주제: [주제 입력])

[Abstract 전문 붙여넣기]
```

방법 B — 여러 논문을 비교 정리

5편 이상의 논문을 읽었다면, 각 논문의 핵심 정보를 AI에게 주고 비교표를 만들게 한다.

프롬프트 — 문헌 비교표 생성
```
아래는 내가 읽은 논문들의 핵심 정보입니다. 이를 비교 표로 만들어 주세요.

논문 1: [저자, 연도, 연구 설계, 대상, 주요 결과]
논문 2: [저자, 연도, 연구 설계, 대상, 주요 결과]
논문 3: [저자, 연도, 연구 설계, 대상, 주요 결과]

비교 항목: 연구 설계, 표본 수, 주요 결과, 한계점, 내 연구와의 관련성
표 형식으로 정리하고, 마지막에 이 문헌들에서 공통으로 드러나는 연구 공백을 2~3줄로 요약해 주세요.
```

| AI 활용 방식 | 장점 | 주의사항 |
|---|---|---|
| Abstract 요약 | 읽는 시간 50% 단축 | 수치 오류 가능 — 원문 대조 필수 |
| 비교표 생성 | 논문 간 패턴 시각화 | 입력 정보가 정확해야 출력도 정확 |
| 연구 공백 분석 | 논의 섹션 방향 잡기 | AI의 의견은 출발점, 검증 필요 |

---

### 단계 3 — 가설 및 논리 구조 정리

Introduction과 Discussion은 논문에서 가장 쓰기 어려운 부분이다. "왜 이 연구가 필요한가"와 "결과가 무엇을 의미하는가"를 논리적으로 서술해야 한다.

프롬프트 — Introduction 논리 구조 설계
```
내 연구의 배경을 정리해 주세요.

연구 주제: [주제]
연구 배경 (내가 아는 것): [간략하게 서술]
연구 목적: [연구 질문]

Introduction을 위한 논리 흐름을 3단계로 구성해 주세요:
1. 큰 그림 (전체 질환/문제의 중요성)
2. 기존 연구의 현황과 한계
3. 내 연구의 필요성과 목적

각 단계를 2~3문장으로 예시 작성해 주세요. (한국어로)
```

프롬프트 — Discussion 논리 전개 보조
```
내 연구 결과를 바탕으로 Discussion 초안을 잡는 데 도움을 주세요.

주요 결과: [결과 요약 — 수치 포함]
예상했던 결과와의 차이: [있다면 서술]
관련 선행 연구: [비교할 논문 1~2편 요약]

Discussion 구성을 위한 질문 5가지를 만들어 주세요.
(예: "이 결과가 기존 연구와 다른 이유는?", "이 결과가 임상에 어떤 의미인가?")
그리고 각 질문에 대해 내 데이터를 바탕으로 답변 방향을 1~2문장씩 제안해 주세요.
```

---

### 단계 4 — 초안 작성: 빈 화면 공포 극복하기

논문을 처음부터 완성된 형태로 쓰려고 하면 막힌다. AI와 함께하는 초안 작성의 핵심은 "일단 쓰고, 나중에 다듬는다"는 원칙이다.

프롬프트 — Methods 섹션 초안
```
아래 정보를 바탕으로 논문 Methods 섹션 초안을 작성해 주세요.

연구 설계: [후향적 코호트 / 전향적 / 단면 연구 등]
연구 기관: [기관 유형 — 3차 병원 등]
연구 기간: [기간]
대상 환자: [포함 기준 / 제외 기준]
수집한 변수: [나열]
통계 방법: [사용한 방법 — t-test, logistic regression 등]
IRB 승인 여부: [승인 번호 또는 면제 여부]

STROBE/CONSORT 형식을 참고해 Methods를 단락으로 작성해 주세요.
초안이므로 [괄호]로 내가 채울 부분을 표시해 주세요.
```

프롬프트 — Results 섹션 표 + 설명 초안
```
아래 통계 결과를 바탕으로 Results 섹션을 작성해 주세요.

Table 1 (기본 특성): [데이터 붙여넣기 또는 요약]
Table 2 (주요 분석): [데이터 붙여넣기 또는 요약]

작성 조건:
- 표 수치를 직접 언급하되 반복 서술은 피할 것
- 통계적 유의성 표현은 "p=0.03" 형태로
- 임상적 의의 해석은 Results에 포함하지 말 것 (Discussion용)
- 영어로 작성해 주세요 (저널 제출용)
```

초안은 시작이지 완성이 아니다. AI가 생성한 문장은 반드시 직접 읽고, 수치를 대조하고, 자신의 언어로 다듬어야 한다.

---

### 단계 5 — 영어 교정: Native-level 문장으로

논문 영어는 일반 영어와 다르다. 학술적 표현, 수동태 관습, 특정 동사 선택 — 이 부분에서 AI는 매우 강력하다.

프롬프트 — 문법 및 표현 교정
```
아래 문단을 학술 논문 스타일로 교정해 주세요.

조건:
- 저널: [목표 저널명 또는 분야 — 예: 내과 계열 SCI 저널]
- 문체: 능동태와 수동태를 적절히 혼합, 간결하게
- 한국어 직역투 표현을 자연스러운 영어로
- 교정 후 원문과 교정본을 나란히 보여주고,
  주요 변경 사항에 대해 이유를 간략히 설명해 주세요.

[교정할 문단 붙여넣기]
```

프롬프트 — 표현 선택지 요청
```
아래 문장을 3가지 다른 방식으로 표현해 주세요.
각각 뉘앙스 차이(강도, 확실성, 격식 수준)를 설명해 주세요.

원문: [문장 입력]
```

프롬프트 — Abstract 전체 교정
```
아래 논문 Abstract를 교정해 주세요.

목표 글자 수: 250단어 이내 (저널 요구 사항)
섹션 구조: Background / Methods / Results / Conclusions
현재 약점이라고 생각하는 부분: [예: Results 수치 표현이 어색함]

교정본을 먼저 제시하고, 섹션별로 주요 변경 이유를 설명해 주세요.

[Abstract 전문 붙여넣기]
```

### 윤리 및 주의사항 — 반드시 알아야 할 3가지

AI 논문 작업에서 가장 중요한 부분이다. 이 섹션을 건너뛰면 투고 거절 또는 연구 윤리 문제로 이어질 수 있다.

① AI 활용 공시 (Disclosure)

대부분의 국제 저널(NEJM, JAMA, Lancet, BMJ 등)은 AI 도구 활용 사실을 Methods 또는 Acknowledgements에 명시하도록 요구한다. 예시 문구:

> "ChatGPT (OpenAI) and Claude (Anthropic) were used to assist with manuscript editing and language revision. The authors take full responsibility for the accuracy and integrity of all content."

저널별로 정책이 다르므로 Author Guidelines를 반드시 확인하라. AI를 저자(author)로 등재하는 것은 허용하지 않는 것이 현재 국제 표준이다.

② 데이터와 환자 정보 입력 금지

실제 환자 데이터(이름, 병록번호, 날짜 등)를 AI에게 직접 입력하면 개인정보보호법 및 병원 정책 위반이 될 수 있다. 익명화 또는 집계 통계만 사용하라.

③ 환각(Hallucination) 검증

AI는 존재하지 않는 논문을 인용하거나, 수치를 변형해 제시하는 경우가 있다. 규칙은 단순하다:

- AI가 언급한 모든 참고문헌은 PubMed 또는 Google Scholar에서 직접 확인
- 수치는 원본 데이터와 100% 대조
- AI의 표현이 아무리 자연스럽더라도, 의미가 자신이 의도한 바와 일치하는지 검토

IRB와 저자권

- 연구 데이터 수집 전 반드시 IRB 심의를 거쳐야 한다 (후향적 연구의 경우 면제 여부 확인)
- AI는 저자가 될 수 없다. 논문의 책임은 모두 인간 저자에게 있다
- 공동저자 기준은 ICMJE 기준 (개념화, 데이터 수집, 해석, 최종 승인)을 따르라

## 핵심 정리

- AI는 논문의 막힌 부분을 뚫는 도구다 — 주제 탐색부터 영어 교정까지 5단계 모두 활용 가능하다
- 구체적인 프롬프트가 핵심이다 — 연구 설계, 목표 저널, 데이터 특성을 함께 입력할수록 출력이 정확해진다
- AI가 생성한 내용은 반드시 검증해야 한다 — 인용 논문, 수치, 표현 모두
- AI 활용 공시는 선택이 아니라 의무다 — 저널 Author Guidelines를 미리 확인하라
- 환자 식별 정보를 AI에 입력하지 않는 것은 기본 원칙이다

## 임상 적용

| 단계 | AI 활용법 | 주의사항 |
|---|---|---|
| ① 주제 탐색 | PICO 질문 구체화, 연구 공백 탐색 | AI 제안 주제는 검색으로 선행 연구 확인 |
| ② 문헌 검토 | Abstract 요약, 논문 비교표 생성 | 수치 오류 가능 — 원문 대조 필수 |
| ③ 가설 정리 | Introduction·Discussion 논리 구조 설계 | AI의 논리 흐름은 방향 제안, 최종 판단은 본인 |
| ④ 초안 작성 | Methods·Results 초안, 빈 화면 극복 | AI 초안을 그대로 제출 금지 — 반드시 자신의 언어로 재작성 |
| ⑤ 영어 교정 | 문법 교정, 표현 선택지, Abstract 교정 | 저널별 AI 공시 정책 확인 후 Acknowledgements 기재 |

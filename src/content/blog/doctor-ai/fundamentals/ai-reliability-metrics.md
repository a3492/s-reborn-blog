## 한줄 요약
AI 논문에서 "정확도 99%"를 봤을 때 안심하면 안 된다 — 불균형 데이터에서 99%는 그냥 "항상 음성"이라고 답하는 것과 같을 수 있다.

## 본문

### 민감도와 특이도: 임상 맥락이 핵심

두 개의 AI가 있다:
- AI-A: 민감도 99%, 특이도 60%
- AI-B: 민감도 70%, 특이도 98%

어느 AI가 더 좋은가? 이 질문에 맥락 없이 답할 수 없다. 임상 목적에 따라 완전히 다른 선택이 된다.

#### 민감도 (Sensitivity) = 진짜 환자를 찾는 능력

```
민감도 = TP / (TP + FN)
       = 실제 환자 중 AI가 환자라고 맞춘 비율

예시: 결핵 검사
  실제 결핵 환자 100명
  AI가 양성으로 판정: 94명 (TP)
  AI가 음성으로 판정 (놓침): 6명 (FN)
  민감도 = 94 / (94 + 6) = 94%
```

민감도가 중요한 상황: 놓치면 안 되는 질환
- 전염성 질환 (결핵, COVID-19 스크리닝)
- 암 조기 발견 (유방암, 자궁경부암 스크리닝)
- 패혈증 조기 경보

민감도가 낮으면 → 많은 환자를 놓친다 → 위음성(False Negative)이 많다

#### 특이도 (Specificity) = 건강한 사람을 구별하는 능력

```
특이도 = TN / (TN + FP)
       = 실제 정상인 중 AI가 정상이라고 맞춘 비율

예시: 결핵 검사
  실제 정상인 900명
  AI가 음성으로 판정: 720명 (TN)
  AI가 양성으로 판정 (과잉알람): 180명 (FP)
  특이도 = 720 / (720 + 180) = 80%
```

특이도가 중요한 상황: 불필요한 후속 검사와 환자 불안을 줄여야 할 때
- 확진 검사 (생검 전 스크리닝)
- 응급 불필요 처치 결정

### PPV와 NPV: 유병률이 바꾸는 실제 임상 의미

민감도와 특이도는 모델의 고정된 성능이다. 그러나 의사가 실제로 알고 싶은 것은 "이 환자가 양성이라고 나왔는데, 실제로 환자일 확률이 얼마인가?"다. 이것이 PPV다.

#### PPV와 NPV 계산

```
PPV (양성예측도) = TP / (TP + FP)
                = AI가 양성이라 했을 때 실제 양성일 확률

NPV (음성예측도) = TN / (TN + FN)
                = AI가 음성이라 했을 때 실제 음성일 확률
```

중요: PPV는 유병률(prevalence)에 따라 크게 달라진다.

```
[같은 AI, 다른 유병률]

AI 성능: 민감도 95%, 특이도 90%

시나리오 A: 고위험군 검사 (유병률 10%)
  1000명 중 환자 100명, 정상 900명
  TP = 95, FP = 90
  PPV = 95 / (95 + 90) = 51.4%
  → 양성이 나와도 실제 환자일 확률 51%

시나리오 B: 일반 건강검진 (유병률 0.5%)
  1000명 중 환자 5명, 정상 995명
  TP = 4.75 ≈ 5, FP = 99.5 ≈ 100
  PPV = 5 / (5 + 100) = 4.5%
  → 양성이 나와도 실제 환자일 확률 4.5%
```

일반 건강검진에서 AI가 "양성"이라고 해도 실제 환자일 확률이 4.5%라는 것이다. 나머지 95.5%는 불필요한 추가 검사를 받게 된다. 이것이 스크리닝 검사에서 과잉진단 문제가 발생하는 이유다.

### ROC Curve와 AUC: 임계값의 유연성

ROC(Receiver Operating Characteristic) Curve는 임계값(threshold)을 바꿔가며 민감도-특이도의 트레이드오프를 나타낸 그래프다.

```
ROC Curve 읽는 법:

y축: 민감도 (TPR, True Positive Rate)
x축: 1-특이도 (FPR, False Positive Rate)

왼쪽 상단으로 갈수록 좋은 모델
  → (0, 1) 지점 = 완벽한 모델
  → 대각선 = 무작위 추측 (AUC 0.5)

AUC (Area Under Curve):
  0.9 이상 = 우수
  0.8-0.9  = 양호
  0.7-0.8  = 보통
  0.5-0.7  = 미흡
```

AUC 0.9의 실제 임상 의미:

AUC 0.9는 "무작위로 뽑은 환자 1명과 정상인 1명을 보여줬을 때, AI가 환자를 더 높게 점수를 매길 확률이 90%"라는 의미다. 이것은 상당히 좋은 성능이지만, 여전히 10%는 순서가 바뀐다.

### 유방암 스크리닝 AI 실제 사례

2020년 Nature에 발표된 Google Health 연구(McKinney et al.)에서 유방암 스크리닝 AI의 성능:

| 지표 | AI 단독 | 방사선과 전문의 1명 | AI + 전문의 |
|------|---------|------------------|------------|
| 민감도 | 89.1% | 88.7% | - |
| 특이도 | 92.4% | 87.2% | - |
| AUC | 0.895 | 0.885 | - |
| 위양성률 감소 | - | - | -5.7% |
| 위음성률 감소 | - | - | -9.4% |

이 숫자만 보면 AI가 방사선과 전문의와 비슷하거나 우월해 보인다. 그러나 논문이 발표된 후 여러 비판이 있었다:

1. 데이터 편향: 영국과 미국 데이터로만 학습 — 다른 인구집단에서 성능 미검증
2. 임계값 선택: AUC가 높아도 실제 임상에서 어떤 임계값을 쓰느냐에 따라 성능 차이 큼
3. 비교 기준: 전문의 1명과 비교 — 실제 현장은 2명이 독립 판독

민감도 94%라는 수치는 나머지 6%를 놓친다는 의미이기도 하다. 유방암 스크리닝에서 6%의 위음성은 연간 수십만 명의 스크리닝 대상자를 고려하면 상당한 수이다.

### 정확도(Accuracy) 함정: 불균형 데이터

```
[불균형 데이터의 정확도 함정]

희귀 질환 스크리닝:
  데이터 구성: 환자 100명 + 정상 9,900명 = 10,000명
  유병률 = 1%

나쁜 AI (항상 "정상"이라고만 답함):
  TP = 0, TN = 9,900
  정확도 = 9,900 / 10,000 = 99%

좋은 AI (실제로 구별 잘 함):
  TP = 90, FP = 200, TN = 9,700, FN = 10
  정확도 = (90 + 9,700) / 10,000 = 97.9%

→ "정확도 99%짜리 AI"가 실제로 더 나쁜 AI
```

이 때문에 의료 AI 평가에서 단순 정확도(Accuracy)는 의미 없는 지표다. 민감도, 특이도, F1-score, AUC를 함께 봐야 한다.

### 주요 의료 AI 제품 성능 비교

| AI 제품 | 적용 분야 | 민감도 | 특이도 | AUC | 비고 |
|---------|----------|--------|--------|-----|------|
| 루닛 INSIGHT CXR | 흉부 X-ray (폐결절) | 97.8% | 96.0% | 0.985 | 식약처 허가, 2022 |
| Google DeepMind | 당뇨망막병증 | 90.0% | 98.0% | - | Lancet 2019 |
| IDx-DR | 당뇨망막병증 | 87.2% | 90.7% | 0.931 | FDA De Novo, 2018 |
| Arterys Cardio AI | 심장 MRI 분석 | 96.0% | 96.0% | - | FDA 510k |
| Viz.ai ContaCT | 뇌졸중 LVO 감지 | 90.9% | 93.1% | - | FDA De Novo, 2018 |

주의: 이 수치들은 각 회사의 검증 데이터 기준이다. 실제 임상 환경(real-world performance)에서는 달라질 수 있다.

## 핵심 정리
- 민감도 = 환자를 찾는 능력 (높을수록 덜 놓침) / 특이도 = 정상인 구별 능력 (높을수록 과잉알람 적음)
- PPV는 유병률에 따라 크게 달라진다 — 같은 AI도 건강검진에서는 양성예측도가 매우 낮을 수 있다
- AUC 0.9는 우수한 성능이지만 임계값 선택에 따라 임상 성능이 달라진다
- "정확도 99%"는 불균형 데이터에서 아무것도 하지 않는 것과 같을 수 있다

## 지표 선택 가이드

| 임상 목적 | 우선 지표 | 이유 |
|----------|----------|------|
| 🔴 놓치면 안 되는 질환 스크리닝 | 민감도, NPV | 위음성 최소화 |
| 🔴 불필요한 처치 방지 | 특이도, PPV | 위양성 최소화 |
| 🟡 전반적 모델 성능 평가 | AUC | 임계값 독립적 평가 |
| 🟡 불균형 데이터 | F1-score, AUC | 정확도 함정 회피 |
| 🟢 균형 데이터 일반 평가 | 정확도, AUC | 종합 성능 |

## 한줄 요약
AI 모델이 업데이트되면 같은 환자 데이터에 같은 질문을 해도 답이 달라질 수 있다 — 의료에서 이것은 단순한 불편이 아니라 안전 문제다.

## 본문

### 같은 질문, 다른 답 — 버전 차이의 현실

2023년 NEJM AI에 발표된 연구(Nori et al., Microsoft)에서 GPT-4가 USMLE(미국 의사 면허 시험)에서 86.7%를 달성했다고 보고했다. 그런데 같은 테스트를 GPT-3.5로 하면 53.6%였다. 동일한 질문, 다른 버전, 전혀 다른 답이다.

임상에서 이것이 의미하는 바:

```
[같은 질문, 버전별 답변 비교]

질문: "심방세동 환자에서 와파린 대신 DOAC 선택 기준은?"

GPT-3.5 답변 (2023년 초):
"CHA₂DS₂-VASc 점수 2점 이상에서 항응고 치료를 권고하며,
 신기능 정상 시 DOAC이 선호됩니다. 단, 기계판막이나
 중등도-중증 승모판막 협착에서는 와파린이 권고됩니다."

GPT-4 답변 (같은 시기):
"CHA₂DS₂-VASc 2점 이상(남성)에서 항응고 치료 권고.
 DOAC 선택 시 신기능(CrCl), 상호작용, 환자 순응도 고려.
 2023 ESC 가이드라인에서 DOAC이 와파린보다 우선 권고.
 기계판막, 중등도 이상 승모판 협착: 와파린 유지."

GPT-4o 답변 (2024년):
[위 내용 + 에독사반 신규 적응증 추가 + 항응고 역전제 정보 포함]
```

각 버전이 "틀린" 것이 아닐 수 있다. 그러나 임상에서 어떤 버전을 사용했는지에 따라 처방이 달라질 수 있고, 추후 의무기록 검토 시 "당시 어떤 AI를 사용했는가"가 문제가 된다.

### GPT-3.5 → GPT-4 → GPT-4o 주요 변화

| 버전 | 출시 | 주요 변화 | 의료 영향 |
|------|------|----------|----------|
| GPT-3.5 | 2022.11 | 기본 대화 가능 | USMLE 53.6%, 단순 설명 수준 |
| GPT-4 | 2023.3 | 추론 능력 대폭 향상 | USMLE 86.7%, 복잡한 임상 추론 가능 |
| GPT-4 Turbo | 2023.11 | 컨텍스트 128K, 지식 업데이트 | 더 긴 의무기록 처리 가능 |
| GPT-4o | 2024.5 | 멀티모달(이미지 포함) | 영상·표 포함 문서 해석 가능 |
| GPT-4o mini | 2024.7 | 경량화, 비용 절감 | 성능 일부 저하 — 버전 확인 필수 |

문제는 OpenAI, Anthropic, Google이 **자동으로 모델을 업데이트**한다는 것이다. 별도 설정 없이 API를 사용하면, 어제까지 GPT-4를 쓰다가 오늘 GPT-4o로 바뀌어 있을 수 있다.

### 병원에서 AI 모델 버전 관리가 필요한 이유

**1. 재현성 (Reproducibility)**

의무기록은 법적 문서다. AI가 진단 보조나 치료 결정에 관여했다면, "어떤 AI의 어떤 버전을 사용했는가"가 기록에 남아야 한다. 버전이 자동 업데이트되면 동일한 조건을 재현할 수 없다.

**2. 검증된 성능 기준**

병원이 특정 AI 도구를 도입할 때 성능 검증(validation)을 한다. GPT-4로 검증했다면, 그 결과는 GPT-4o에 그대로 적용되지 않는다.

**3. 예상치 못한 성능 저하**

```
[실제 보고된 사례 유형]

GPT-4 Turbo 출시 후 일부 의료 기관에서:
- 이전 GPT-4보다 특정 임상 추론 과제에서 정확도 하락
- 이유: 모델 경량화 과정에서 의료 특화 데이터 비중 변화
- 기관은 자동 업데이트로 인해 변화를 즉시 인지하지 못했음
```

### 모델 버전 고정: API 파라미터

AI API를 사용하는 의료 기관이라면 버전을 명시적으로 고정할 수 있다.

```python
# OpenAI API — 버전 고정 방법
from openai import OpenAI

client = OpenAI()

response = client.chat.completions.create(
    model="gpt-4-0613",       # 날짜 포함 버전명으로 고정
    # model="gpt-4"는 자동 업데이트됨 — 사용 금지
    messages=[
        {"role": "user", "content": "환자 질문..."}
    ]
)

# 버전 기록을 의무기록 시스템에 저장
used_model = response.model   # 실제 사용된 모델명 반환
# → 로그에 저장: "2024-03-15 14:32 / gpt-4-0613 / 처방 보조"
```

```python
# Anthropic Claude — 버전 고정
import anthropic

client = anthropic.Anthropic()

message = client.messages.create(
    model="claude-3-opus-20240229",   # 날짜 포함 버전으로 고정
    # model="claude-3-opus"는 최신 버전으로 자동 변경될 수 있음
    max_tokens=1024,
    messages=[{"role": "user", "content": "..."}]
)
```

버전명에 날짜가 포함된 형태(예: `gpt-4-0613`, `claude-3-opus-20240229`)를 사용하면 명시적 버전 고정이 된다.

### 의료기기 소프트웨어(SaMD) 변경 시 재허가 이슈

AI 기반 의료기기 소프트웨어는 국내에서 **식약처 SaMD(Software as a Medical Device)** 규정의 적용을 받는다.

```
[식약처 의료기기 소프트웨어 변경 허가 기준 (2023)]

경미한 변경 (신고만 필요):
- UI 개선, 버그 수정
- 알고리즘 경미한 수정으로 성능 동등 이상

중요한 변경 (변경허가 필요):
- 사용 목적 변경
- 알고리즘 구조 변경
- 학습 데이터 대폭 변경으로 성능 변화
- AI 모델 버전 업그레이드로 임상 성능 변화

→ 모델을 GPT-4에서 GPT-4o로 업그레이드하면
  → 성능 변화 여부 평가 후
  → 중요한 변경 해당 시 변경허가 신청 필요
```

FDA도 2021년 AI/ML 기반 SaMD 행동 계획(Action Plan)에서 **Predetermined Change Control Plan (PCCP)**을 도입했다. 모델 업데이트를 사전에 계획하고 허가받는 방식이다.

즉, 병원이 허가받은 AI 의료기기의 백엔드 모델을 무단으로 업그레이드하면 **의료기기법 위반**이 될 수 있다.

### 실전 관리 방안

```
[병원 AI 도구 버전 관리 체크리스트]

도입 단계:
□ 사용할 AI 도구의 현재 모델 버전을 문서화
□ 버전 고정 가능 여부 확인 (API 접근 여부)
□ 의료기기 허가 여부 확인 (식약처 SaMD 해당 시)

운영 단계:
□ 자동 업데이트 알림 설정 (이메일, Slack 등)
□ 업데이트 발생 시 성능 재검증 프로토콜 수립
□ AI 사용 로그에 모델 버전 자동 기록

업데이트 발생 시:
□ 업데이트 전후 성능 비교 테스트
□ 의료진에게 변경 사항 공지
□ SaMD 해당 시 식약처 변경신고/허가 검토
```

## 핵심 정리
- 같은 AI 도구도 버전이 바뀌면 같은 질문에 다른 답을 줄 수 있다
- 자동 업데이트 환경에서 병원은 어떤 버전을 사용했는지 추적해야 한다
- API 사용 시 날짜 포함 버전명(gpt-4-0613 등)으로 명시적 고정 가능
- SaMD로 허가된 AI 의료기기의 모델 업그레이드는 식약처 신고/허가 대상일 수 있다

## 한줄 요약
의료 AI를 임상에서 사용하려면 "좋은 AI인가"만으로는 부족하다 — "허가된 AI인가", "어떤 용도로 허가받았나"를 먼저 확인해야 한다.

## 본문

### 왜 의료 AI 규제를 알아야 하는가

임상의가 의료 AI 규제를 알아야 하는 이유는 세 가지다:

1. 법적 책임: 허가받지 않은 AI를 진단·치료 결정에 사용하면 의사에게 책임이 돌아온다
2. 적용 범위 확인: 허가 용도 이외의 사용은 off-label 적용이 된다
3. 성능 신뢰 기준: 허가 여부는 최소 성능 기준을 통과했다는 의미다

### 3개국 규제 체계 비교

| 구분 | 한국 | 미국 | EU |
|------|------|------|----|
| 주관 기관 | 식품의약품안전처(MFDS) | FDA | EMA + 각국 Notified Body |
| 근거 법령 | 의료기기법 | FD&C Act | MDR 2017/745 + EU AI Act |
| AI 분류 | SaMD (의료기기 소프트웨어) | SaMD | Software as Medical Device |
| 허가 트랙 | 등급별 인허가 | 510k / De Novo / PMA | CE Marking |
| AI 전용 규제 | AI·빅데이터 의료기기 가이드라인 (2019) | AI/ML-based SaMD Action Plan (2021) | EU AI Act (2024) |
| 허가 소요 | 6개월~2년 | 3개월~3년 | 1~3년 |

### 한국: 식약처 SaMD 규제

#### 의료기기 등급 분류

```
의료기기법 제6조 — 4등급 분류 체계

1등급 (경미한 위해): 일반 의료용품
  예: 설압자, 장갑

2등급 (낮은 위험): 신고 또는 인증
  예: 혈압계, 청진기

3등급 (중간 위험): 허가 필요
  예: 인공관절, 진단용 소프트웨어 일부

4등급 (높은 위험): 엄격한 허가
  예: 심장판막, 생명유지 장치
```

AI 기반 진단 소프트웨어는 용도와 위험도에 따라 2~4등급으로 분류된다. 대부분의 의료 AI 진단 보조 소프트웨어는 3등급이다.

#### 혁신의료기기 제도 (2019년 시행)

```
혁신의료기기 지정 요건:
  - 인공지능·빅데이터 활용
  - 기존 의료기기 대비 성능·안전성 현저 개선
  - 또는 새로운 임상적 유용성 제공

혜택:
  - 우선 심사 (검토 기간 단축)
  - 개발 단계부터 식약처와 사전 상담
  - 시장 출시 후 임상 데이터 수집 허용 (조건부 허가)

현황 (2024 기준):
  지정 건수: 누적 200건 이상
  주요 분야: AI 영상진단, 디지털 치료기기
```

#### 실제 사례: 루닛 INSIGHT CXR 허가 과정

```
루닛 INSIGHT CXR — 흉부 X-ray AI 판독 보조 소프트웨어

허가 신청: 2018년
허가 완료: 2020년 (식약처 3등급 의료기기)
허가 용도: 흉부 X-ray에서 결절·종괴 검출 보조

허가 과정:
  1. 임상 시험 설계 (식약처 사전 상담)
     → 다기관 후향적 임상: 4개 의료기관, 5,000장 X-ray
  2. 성능 기준 충족 검증
     → 민감도 97.8%, 특이도 96.0% (독립 검증 데이터)
  3. 소프트웨어 생명주기 관리 문서 제출
     → IEC 62304 (의료기기 소프트웨어 개발 표준) 준수
  4. 식약처 심사 → 3등급 허가

해외 진출:
  CE 마킹 (유럽): 2020년
  FDA 510k (미국): 2020년
```

### 미국: FDA SaMD 규제

#### 허가 트랙 비교

```
510k (Pre-market Notification):
  요건: 기존 허가된 유사 기기 (predicate)와 동등한 성능 입증
  소요: 평균 3-6개월
  적용: 대부분의 AI 진단 보조 소프트웨어

De Novo:
  요건: 유사 predicate 없는 새로운 기기
  소요: 평균 12-24개월
  적용: IDx-DR (당뇨망막병증 자율 진단 AI, 2018 — FDA 최초 AI De Novo)

PMA (Premarket Approval):
  요건: 임상 시험 데이터로 안전성·유효성 직접 입증
  소요: 1-3년
  적용: 심장판막, 고위험 임플란트 (AI에는 드물게 적용)
```

#### FDA AI/ML 프레임워크 (2021)

FDA는 2021년 AI/ML 기반 SaMD 행동 계획(Action Plan)에서 Predetermined Change Control Plan (PCCP)을 도입했다.

```
PCCP (사전 변경 관리 계획):
  기존 문제: AI 모델이 업데이트될 때마다 재허가 신청 필요
           → 빠른 AI 발전 속도에 규제가 따라가기 어려움

PCCP 해결책:
  - 허가 신청 시 미래 변경 계획도 함께 제출
  - "이런 범위 내의 변경은 재허가 없이 허용"을 사전 승인
  - 변경 후 성능 모니터링 데이터 제출 의무

예시:
  허가 당시 PCCP에 "학습 데이터 20% 추가 시 성능 동등 이상이면
  재허가 불필요" 명시 → 해당 조건 충족 시 재허가 없이 업데이트 가능
```

### EU: AI Act + MDR 이중 규제

EU는 의료 AI에 두 가지 규제가 겹쳐 적용된다.

#### MDR (Medical Device Regulation, 2017/745)

```
CE Marking 획득 과정:
  1. 의료기기 분류 (Class I/IIa/IIb/III)
  2. 적합성 평가 (Notified Body 심사)
  3. 기술 파일 제출 (임상 근거, 성능 데이터)
  4. CE 마크 발급

AI 진단 소프트웨어: 주로 Class IIa 또는 IIb
```

#### EU AI Act (2024년 발효)

```
EU AI Act의 의료 AI 분류:

고위험 AI (High-risk AI) — 의무 요건 적용:
  - 의료 기기 소프트웨어 (MDR/IVDR 범위)
  - 환자 위험 관리, 응급 의료 자원 배분

요구 사항:
  □ 위험 관리 시스템
  □ 데이터 거버넌스 (학습 데이터 관리)
  □ 기술 문서화
  □ 투명성 (설명가능성)
  □ 인간 감독 (Human Oversight)
  □ 견고성·정확성·사이버보안

금지 AI (Prohibited AI):
  - 사회적 점수 시스템
  - 의식에 영향 미치는 AI 조작
  (의료 AI는 해당 없음)
```

이중 규제의 실무 부담:

```
유럽에서 의료 AI 출시하려면:

MDR 요건 + AI Act 요건 동시 충족
  → 기술 문서 두 세트
  → Notified Body + AI Act 감독 기관
  → 비용과 시간 대폭 증가

업계 반응:
  → 일부 기업이 유럽 시장 진출 지연
  → 미국·한국 먼저 출시 후 유럽 진출하는 전략 증가
```

### 3개국 규제 속도 비교

| 항목 | 한국 | 미국 | EU |
|------|------|------|----|
| 혁신 의료기기 우선 트랙 | ✅ 혁신의료기기 | ✅ Breakthrough Device | △ 없음 (Fast Track 논의 중) |
| AI 전용 가이드라인 | ✅ 2019년 | ✅ 2021년 | ✅ AI Act 2024 |
| 모델 업데이트 허용 | △ 변경신고 필요 | ✅ PCCP로 사전 계획 가능 | 🔴 MDR 변경 심사 엄격 |
| 시판 후 성능 모니터링 | 의무 | 의무 | 의무 |
| 전반적 규제 유연성 | 중간 | 높음 | 낮음 |

### 의사가 현장에서 확인해야 할 것

```
[의료 AI 도입 전 체크리스트]

국내 사용 시:
□ 식약처 의료기기 허가 번호 확인
  → 의료기기 전자민원창구 (udiportal.mfds.go.kr) 검색
□ 허가 용도(Intended Use) 확인
  → "진단 보조"인지 "진단"인지 구분
□ 등급 확인 (3등급 이상이면 안전성 심사 통과)

수입 AI 제품 사용 시:
□ 국내 허가 여부 별도 확인
  → FDA 허가 = 국내 허가 아님
  → CE 마킹 = 국내 허가 아님
□ 국내 판매업 허가 확인 (의료기기법 제15조)

임상 적용 범위:
□ AI가 허가받은 용도에 맞게 사용하고 있는지
□ 허가 외 사용(off-label) 시 환자 동의 및 기록 필요
```

## 핵심 정리
- 한국: 식약처 SaMD 3등급 허가, 혁신의료기기 우선 심사 트랙 운영
- 미국: FDA 510k/De Novo, PCCP로 모델 업데이트 유연성 확보
- EU: MDR + AI Act 이중 규제로 가장 엄격 — 진입 장벽 높음
- 의사는 임상에서 사용하는 AI의 국내 허가 여부와 허가 용도를 반드시 확인해야 한다

## 규제 적용 범위 요약

| 위험도 | 상황 | 대응 |
|:------:|------|------|
| 🔴 높음 | 국내 미허가 AI를 진단 결정에 사용 | 사용 금지 또는 연구 목적 IRB 필요 |
| 🔴 높음 | 허가 외 용도(off-label) AI 사용 | 환자 동의 + 의무기록 명시 |
| 🟡 중간 | 허가된 AI를 권고 용도 내에서 사용 | 성능 모니터링, 버전 기록 |
| 🟢 낮음 | 교육·행정 목적 비허가 AI 사용 | 진단·처방에 직접 사용 금지 조건 하에 가능 |

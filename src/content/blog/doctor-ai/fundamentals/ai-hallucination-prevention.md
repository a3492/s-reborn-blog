## 한줄 요약
AI 답변을 그대로 처방에 적용한 당직 의사가 있었다 — 5단계 검증 프로토콜이 있었다면 막을 수 있었다.

## 본문

### 검증 없이 AI를 사용하면 생기는 일

2023년 미국의 한 사례에서 당직 전공의가 야간에 희귀 진균 감염 환자의 항진균제 용량을 ChatGPT에 물었다. AI는 자신 있게 답했고, 피곤한 당직 환경에서 별도 확인 없이 처방이 이루어졌다. 용량은 표준 용량의 2배였다.

환자는 간독성으로 ICU에 입원했다.

이것은 AI가 특별히 나쁜 게 아니다. 검증 프로토콜 없이 AI를 도구로 사용하는 방식이 문제였다. 청진기로 맥박을 재지 않듯, AI 답변을 그대로 처방에 쓰지 않아야 한다.

아래는 의료 현장에서 실전으로 적용할 수 있는 5단계 검증 프로토콜이다.

### 단계 1: 숫자와 수치는 무조건 교차확인

AI가 제시하는 모든 숫자에는 빨간 깃발을 꽂아야 한다.

위험 수치 목록:
- 약물 용량 (mg, mg/kg, mL)
- 진단 기준값 (혈당, HbA1c, 혈압, eGFR 등)
- 약물 투여 간격 (q8h, q12h 등)
- 금기 기준 (eGFR < 30 등)

```
[검증 루틴]

AI 답변에서 숫자 발견
  ↓
공식 자료로 이동
  - 약물 용량: 병원 약전, UpToDate, MICROMEDEX
  - 진단 기준: 해당 학회 최신 가이드라인 원문
  - 상호작용: Drugs.com, Lexicomp
  ↓
숫자가 일치하면 사용
숫자가 다르면 → 공식 자료 우선
```

실제로 GPT-4도 메트포르민 최대 용량을 "3000mg"으로 답한 사례가 있다. 실제 한국 허가 기준은 2550mg, 신기능에 따라 더 낮아진다.

### 단계 2: 출처를 추가로 물어보기

```
[실전 프롬프트 — 검증용 후속 질문]

"방금 답변의 근거가 되는 논문이나 가이드라인을
 정확한 제목, 저자, 발행 연도, 저널명으로 알려줘.
 출처를 모르면 '확인이 필요하다'고 말해줘."
```

이 질문을 하면 두 가지 일이 벌어진다:

1. AI가 출처를 제시한다 → PubMed나 학회 사이트에서 직접 검색
2. AI가 "확인이 필요합니다"라고 인정한다 → 원본 답변도 재검토

```
[PubMed 검색 방법]

https://pubmed.ncbi.nlm.nih.gov/

검색창에:
  저자 성[au] + 키워드 + 연도[dp]
  예: Williams[au] AND pediatric septic shock AND 2022[dp]

결과 없음 → 해당 논문 환각으로 판단
```

주의: 출처를 물었을 때 AI가 제시하는 논문도 환각일 수 있다. 반드시 직접 검색으로 존재 여부 확인.

### 단계 3: 같은 질문을 다르게 두 번

환각은 일관성이 없다. 실제로 옳은 정보라면 질문 방식이 바뀌어도 일관된 답이 나온다.

```
[동일 내용 재확인 기법]

1차 질문:
"신장 기능 저하(eGFR 35) 환자에서 메트포르민 사용 가능한가?"

2차 질문 (재구성):
"eGFR 35인 2형 당뇨 환자에게 메트포르민을 처방할 때
 주의해야 할 점은? 금기 여부를 포함해 알려줘."

→ 두 답변이 일치하면 신뢰도 ↑
→ 두 답변이 다르면 → 공식 자료로 직접 확인 필수
```

다른 AI 모델(ChatGPT vs Claude vs Gemini)에 같은 질문을 해보는 것도 효과적이다. 모든 모델이 같은 답을 주면 신뢰도가 높아진다.

### 단계 4: RAG 기반 의료 도구 우선 사용

RAG(Retrieval-Augmented Generation)는 실제 문서를 검색해서 그 내용을 기반으로 답변하는 방식이다. 일반 LLM의 "기억에 의존한 예측"보다 환각이 크게 줄어든다.

| 도구 | 특징 | 의료 적합성 |
|------|------|------------|
| Consensus | PubMed 기반 논문 요약, 근거 수준 표시 | 문헌 검색, 가이드라인 확인에 적합 |
| Elicit | 연구 방법론 분석, RCT 추출 | 임상 근거 평가에 적합 |
| UpToDate | 최신 임상 가이드라인 DB | 처방 결정, 용량 확인에 적합 |
| PubMed AI | PubMed 직접 연동 | 논문 검색 자동화 |
| 일반 ChatGPT | 범용 LLM, 문서 연동 없음 | 개념 설명, 교육 초안에만 사용 |

```
[실전 워크플로우]

의료 AI 활용 시:
1. 개념 파악/초안 작성 → 일반 LLM 사용 가능
2. 약물 용량/가이드라인 → UpToDate / MICROMEDEX
3. 문헌 근거 확인 → Consensus / PubMed
4. 최종 처방 결정 → 공식 자료 + 의사 판단
```

### 단계 5: "모르면 모른다고 해" 지시문 추가

LLM은 기본적으로 답을 생성하려는 경향이 있다. 이것을 억제하는 지시문을 프롬프트에 포함시킨다.

```
[시스템 프롬프트 템플릿 — 의료 용도]

당신은 의료 정보를 제공하는 보조 도구입니다.
다음 규칙을 반드시 따르세요:

1. 확실하지 않은 정보는 "확인이 필요합니다"라고 명시
2. 약물 용량, 진단 기준 등 수치는 반드시
   "공식 자료(약전, 가이드라인)로 확인하세요"라는 주석 추가
3. 논문이나 가이드라인을 인용할 때는
   "PubMed에서 직접 검색하세요"라고 안내
4. 추측으로 답하지 말고, 불확실한 경우 솔직히 말할 것
```

이 지시문을 추가했을 때 환각 빈도가 완전히 사라지지는 않지만, AI가 스스로 불확실성을 표시하는 빈도가 증가한다는 연구 결과가 있다 (Kadavath et al., 2022, Anthropic).

### 현장 적용 체크리스트

```
[의료 AI 사용 전 체크리스트]

□ 1. 이 정보는 직접 생명/처방과 관련된 수치인가?
       Yes → 반드시 공식 자료 교차확인
       No → 다음 단계로

□ 2. AI가 수치나 통계를 제시했는가?
       Yes → 출처 질문 → PubMed/약전 검색
       No → 다음 단계로

□ 3. 논문이나 가이드라인을 인용했는가?
       Yes → 제목+저자 PubMed 검색으로 존재 확인
       No → 다음 단계로

□ 4. 같은 내용을 다르게 다시 물어봤는가?
       일치 → 사용 가능 (단, 공식 확인 권고)
       불일치 → 공식 자료로 직접 확인

□ 5. RAG 기반 도구를 사용했는가?
       Consensus/UpToDate 등 사용 → 신뢰도 ↑
       일반 LLM만 사용 → 추가 검증 필수
```

### 검증이 필요 없는 경우

모든 AI 사용에 5단계 검증을 적용하면 오히려 AI 도구의 효율이 떨어진다. 다음 상황에서는 검증 수준을 낮출 수 있다:

- 교육 자료 초안 작성 (최종 내용은 검토)
- 의무기록 정리 형식 개선
- 연구 아이디어 브레인스토밍
- 의학 용어 설명 (환자 교육용 쉬운 표현)

반대로 절대 검증을 생략하면 안 되는 경우:

- 약물 용량, 투여 경로, 간격
- 금기 및 주의 기준 (eGFR, 체중, 나이별)
- 진단 기준 수치
- 논문 인용 (학술 발표, 보고서)

## 핵심 정리
- 당직 환경에서 피로한 의사가 AI 답변을 무검증 적용하면 실제 위해가 발생한다
- 5단계: 수치 교차확인 → 출처 질문 → 재질문 → RAG 도구 사용 → 불확실성 지시문
- RAG 기반 도구(Consensus, UpToDate)가 일반 LLM보다 의료 용도에 훨씬 안전하다
- 모든 AI 사용에 검증이 필요한 것은 아니다 — 위험도에 따라 수준을 조정하라

## 위험도별 검증 강도

| 위험도 | 영역 | 검증 수준 |
|:------:|------|----------|
| 🔴 높음 | 약물 용량·투여 경로 | 공식 자료 필수 교차확인, AI 답변 참고만 |
| 🔴 높음 | 진단 기준·금기 수치 | 학회 가이드라인 원문 확인 |
| 🟡 중간 | 논문 인용·근거 | PubMed 직접 검색으로 존재 확인 |
| 🟡 중간 | 감별진단·임상 전략 | 재질문 + 임상 맥락으로 필터링 |
| 🟢 낮음 | 교육 초안·설명 자료 | 내용 검토 후 수정해서 사용 |

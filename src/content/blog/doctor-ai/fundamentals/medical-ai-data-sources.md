## 한줄 요약
의료 AI가 왜 백인 남성 환자에게 더 잘 작동하는지는 학습 데이터를 보면 답이 나온다.

## 본문

### 데이터가 AI를 만든다

의료 AI의 성능은 알고리즘보다 **데이터의 품질과 다양성**에 더 크게 좌우된다. 같은 딥러닝 아키텍처라도 학습 데이터가 다르면 완전히 다른 AI가 된다.

의사가 학습 데이터를 이해해야 하는 이유:
- AI가 왜 특정 환자군에서 성능이 낮은지 설명 가능
- 어떤 환자에게 AI를 적용하면 위험한지 판단 가능
- AI 논문의 "검증 데이터"가 우리 환자 집단과 같은지 확인 가능

### 주요 공개 의료 데이터셋

#### MIMIC-III / MIMIC-IV — 중환자실 임상 데이터

```
MIMIC (Medical Information Mart for Intensive Care)
개발: MIT Laboratory for Computational Physiology

MIMIC-III (2016):
  - 출처: Beth Israel Deaconess Medical Center (보스턴) ICU
  - 기간: 2001-2012년
  - 환자 수: 약 46,000명 입원 에피소드
  - 포함 정보: 바이탈, 검사 결과, 처방, 진단코드(ICD-9), 의무기록 노트

MIMIC-IV (2023):
  - MIMIC-III 업데이트 버전
  - 환자 수: 약 382,000명
  - 기간: 2008-2022년
  - ICD-10 코드 추가

활용 예시:
  - 패혈증 조기 경보 모델 학습
  - 재원 기간 예측
  - 약물 처방 패턴 분석
```

**한계**: 미국 단일 기관(보스턴 백인 중산층 환자 다수), 영어 의무기록 전용. 한국 환자에게 직접 적용 시 성능 저하 가능.

#### CheXpert — 흉부 X-ray 라벨 데이터

```
CheXpert (Stanford, 2019)
  - 환자 수: 224,316명 (65,240명 고유 환자)
  - 이미지 수: 224,316장 흉부 X-ray
  - 라벨: 14개 병변 (폐렴, 흉막 삼출, 폐부종 등)
  - 라벨링 방법: NLP 자동 추출 → 방사선과 전문의 검증

성능 비준점:
  AUC 0.9 이상 달성한 병변: 심비대, 흉막 삼출, 무기폐
  어려운 병변: 폐렴 (AUC ~0.76), 폐결절 (AUC ~0.71)
```

#### NIH ChestX-ray14 — 대규모 흉부 X-ray

```
NIH ChestX-ray14 (2017)
  - 이미지 수: 112,120장 (30,805명 고유 환자)
  - 라벨: 14개 병변
  - 라벨링: ICD 코드에서 NLP 자동 추출 (방사선과 직접 라벨링 아님)

주요 한계:
  - NLP 자동 라벨링으로 오류율 높음 (~10-20%)
  - 폐렴 라벨 정확도 논란 (실제 세균성/바이러스성 구분 안 됨)
```

#### Google's DeepMind 당뇨망막병증 데이터

```
Kaggle DR Dataset / EyePACS
  - 이미지 수: 88,000장 안저 사진
  - 출처: 미국 당뇨 환자 클리닉
  - 라벨: 0-4단계 (없음/경증/중등/중증/증식성)

DeepMind 자체 데이터 (Lancet 2019):
  - 영국·인도 병원에서 수집
  - 인도 환자 포함 → 다민족 검증
```

### 한국 의료 AI 데이터

#### 건강보험심사평가원(HIRA) 청구 데이터

```
규모: 연간 약 14억 건 진료 청구
포함: 진단코드(KCD), 처방, 시술, 병원 종별
특징: 전국민 커버리지 (세계 최대 수준의 단일 국가 보험 DB)

활용 예시:
  - 희귀질환 환자 실태 파악
  - 약물 처방 패턴 및 부작용 분석
  - 지역별 의료 이용 불균형 연구

한계:
  - 진단 코드 정확도 (보험 청구 목적 코딩 → 임상적 정확성 제한)
  - 영상/검사 결과 데이터 없음 (처방 코드만 있음)
  - 실제 임상 소견 없음
```

#### 국립암센터 (NCC) 암 등록 데이터

```
국가암등록 사업 (1980년 시작)
  - 연간 신규 암환자 약 24만 명 등록
  - 포함: 암 종류, 병기, 치료 방법, 생존율
  - 전국 병원 의무 보고

활용: 암 예후 예측 모델, 치료 효과 분석
한계: 항암제 용량, 부작용 세부 정보 없음
```

### 데이터 품질 문제

#### 라벨링 오류

의료 데이터의 라벨(정답)은 사람이 붙인다. 이 과정에서 오류가 발생한다.

```
[라벨링 오류 유형]

1. 전문가 간 불일치 (Inter-rater Variability)
   폐결절 악성 여부: 방사선과 전문의 2명 간 일치율 ~75%
   피부 병변: 피부과 전문의 3명 간 완전 일치 ~60%

2. NLP 자동 라벨링 오류
   "폐렴 없음"이라는 의무기록 → NLP가 "폐렴" 태그 생성
   → NIH ChestX-ray14에서 이런 역라벨링 오류 보고

3. 생존 편향 (Survival Bias)
   MIMIC: ICU 입원 환자만 포함
   → 경증 환자에게 적용 시 모델 성능 저하
```

#### 인구 편향 (Population Bias)

```
[피부질환 AI의 인종 편향 사례]

Esteva et al. (2017, Nature): 피부암 분류 AI
  학습 데이터: Stanford 병원 환자 (백인 다수)
  성능: 방사선과 전문의와 동등

후속 연구 (Adamson & Smith, 2018, JAMA Dermatology):
  - 어두운 피부색 환자 이미지 성능: 현저히 낮음
  - 원인: 학습 데이터에 어두운 피부 환자 비율 3.4% 미만

→ 한국 환자에게 미국 백인 데이터로 학습된 AI 적용 시
   유사한 편향 발생 가능
```

#### 누락 데이터 (Missing Data)

실제 EMR 데이터는 결측값이 많다. 당직 시간에 기록된 데이터, 외래에서 빠진 검사, 이전 병원에서 이관되지 않은 병력 — 이런 누락이 AI 성능에 영향을 준다.

```
[MIMIC-III 주요 변수 결측률]

eGFR: 약 22% 결측
체중: 약 34% 결측
혈압 (24시간): 약 18% 결측

→ 결측 처리 방식(평균값 대체 vs 다중 대체 vs 제거)에 따라
  AI 성능이 달라짐
```

### 개인정보 보호: HIPAA와 한국 가명처리

#### HIPAA de-identification (미국)

```
HIPAA Safe Harbor 방법:
18개 식별자 제거 필수:
  - 이름, 지리 정보 (우편번호 3자리까지 제거)
  - 날짜 (생년월일 → 연도만, 90세 이상 → 90+로 표시)
  - 전화번호, 이메일, SSN, 의무기록번호
  - 사진, 생체 정보

Expert Determination 방법:
  통계 전문가가 재식별 위험이 매우 낮다고 인증
```

#### 한국 개인정보 보호법 — 가명처리

```
개인정보 보호법 제28조의2 (2020년 시행)
  - 가명처리 후 통계 작성, 연구 목적 사용 가능
  - 재식별 위험을 낮추는 처리 필요

의료 데이터 특례:
  - 보건의료 데이터 활용 가이드라인 (복지부, 2021)
  - 연구 목적 가명 의료데이터 활용 허용
  - IRB 심의 + 데이터 보안 요건 충족 필요

실제 사례:
  건강보험공단 빅데이터 개방: 연구 목적 가명 청구 데이터
  → 가명처리 후 연구자 신청으로 접근 가능
```

### 데이터 불균형이 AI 성능에 미치는 영향

```
[희귀 진단 학습의 현실]

가정:
  흉부 X-ray 100,000장 학습
  구성:
    정상 70,000장 (70%)
    폐렴 20,000장 (20%)
    폐결절 9,000장 (9%)
    희귀 폐질환 A 500장 (0.5%)
    희귀 폐질환 B 100장 (0.1%)

결과:
  → AI는 폐렴, 정상 판독에 매우 강함
  → 희귀 폐질환 A/B 판독 성능 매우 낮음
  → 전체 정확도는 높아도 희귀 질환에서 실패

대응:
  → Oversampling (소수 클래스 반복 학습)
  → Synthetic data 생성 (GAN 활용)
  → 전이학습 (transfer learning)으로 소수 클래스 성능 보완
```

## 핵심 정리
- 의료 AI의 학습 데이터는 MIMIC, CheXpert 등 공개 DB와 병원 EMR, 국가 청구 데이터로 구성된다
- 한국에는 건강보험 청구 데이터(세계 최대 수준)와 국가암등록 데이터가 있으나 임상 세부 정보 부족
- 라벨링 오류, 인구 편향, 결측 데이터가 AI 성능의 숨은 한계를 만든다
- 미국 데이터로 학습된 AI를 한국 환자에게 그대로 적용하면 편향 위험이 있다

## 데이터 품질 체크리스트 (AI 도구 도입 평가 시)

| 확인 항목 | 위험도 | 질문 |
|----------|:------:|------|
| 학습 데이터 인구 구성 | 🔴 높음 | 우리 환자 집단과 비슷한가? |
| 검증 데이터 독립성 | 🔴 높음 | 학습/검증이 같은 병원 데이터인가? |
| 라벨링 방식 | 🟡 중간 | 전문가 직접 라벨링인가, NLP 자동인가? |
| 데이터 수집 기간 | 🟡 중간 | 최신 치료 가이드라인 반영 시기인가? |
| 결측 데이터 처리 | 🟡 중간 | 결측 처리 방법이 논문에 명시되어 있나? |
| 한국 검증 여부 | 🟢 낮음 | 국내 병원 데이터로 재검증되었나? |

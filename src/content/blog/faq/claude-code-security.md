## 한줄 요약
Claude Code가 읽은 파일 내용과 대화는 Anthropic 서버로 전송된다. Anthropic은 이 데이터를 모델 학습에 쓰지 않겠다고 명시하지만, 민감한 정보(비밀번호, 환자 데이터)는 넣지 않는 것이 원칙이다.

## Q: Claude Code를 쓸 때 내 코드가 Anthropic에 저장되나요?

## A: 간단 답변
Claude Code가 읽은 파일과 대화 내용은 Anthropic 서버로 전송된다. API 이용약관에 따라 모델 학습에는 사용되지 않는다고 되어 있지만, 완전한 프라이버시가 보장되는 환경은 아니다. 환자 정보, 비밀번호, 내부 기밀 코드는 Claude Code에 노출하지 않는 것이 안전하다.

## 상세 설명

### 데이터 전송 구조

Claude Code가 파일을 읽으면, 그 내용이 Anthropic API 요청의 일부로 서버에 전송된다.

```
[내 컴퓨터] → [Anthropic API 서버] → [Claude 모델 처리] → [응답 반환]
```

파일 내용이 프롬프트에 포함되어 전송되는 것이다.

### Anthropic의 데이터 정책

**API 이용 시**
- API를 통해 전송된 데이터는 기본적으로 모델 학습에 사용되지 않는다
- 30일 동안 서버에 보관 후 삭제 (정책에 따라 다를 수 있음)
- 콘텐츠 안전 모니터링은 진행될 수 있음

**Enterprise 계약 시**
- 더 강력한 데이터 보호 계약 체결 가능
- 데이터 보관 기간 등 별도 협의 가능

### 넣으면 안 되는 것

| 항목 | 이유 |
|------|------|
| 환자 이름·주민번호 등 개인정보 | 의료법·개인정보보호법 위반 가능 |
| 데이터베이스 비밀번호 | 유출 시 심각한 피해 |
| AWS·GCP 시크릿 키 | 유출 시 무단 과금 발생 |
| 사내 기밀 문서 | 회사 정책 위반 가능 |
| 미발표 연구 데이터 | 지적재산 유출 위험 |

### 의료 환경에서의 주의

의사·임상 연구자가 Claude Code를 쓸 때:
- 환자 데이터가 포함된 파일은 절대 넣지 않는다
- 코드를 분석할 때 테스트 데이터(더미 데이터)를 사용한다
- 익명화된 데이터도 민감 정보가 포함될 수 있어 주의 필요

### 실제로 안전하게 쓰는 방법

**환경변수 파일 제외**
`.env` 파일에 API 키가 있는 경우, Claude Code에 "이 파일은 읽지 마"라고 명시한다.

```
> .env 파일은 무시하고, app.py의 로직만 분석해줘
```

**민감 정보 마스킹**
테스트 데이터를 쓸 때 실제 이름 대신 "홍길동", 실제 번호 대신 "010-0000-0000" 형태를 사용한다.

**코드 리뷰 후 공유**
Claude Code에 넣기 전, 해당 코드에 실제 민감 정보가 하드코딩되어 있지 않은지 확인한다.

## 핵심 정리
- 파일 내용과 대화는 Anthropic 서버로 전송됨
- API 이용 시 모델 학습에는 사용 안 된다고 명시되어 있음
- 환자 정보·비밀번호·시크릿 키는 절대 포함하지 말 것
- 의료 환경에서는 더미 데이터 사용 권장

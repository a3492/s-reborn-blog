## 한줄 요약
의료 AI가 정확한 답변을 하려면 여러 출처(EMR, 가이드라인, 논문)의 정보를 하나의 컨텍스트에 통합해야 하며, 출처 간 충돌을 명시적으로 처리하는 것이 핵심이다.

## 본문

### 단일 출처의 한계

AI에게 하나의 정보 출처만 제공하면 답변에 편향이 생긴다.

- EMR만 제공: 환자 데이터는 있지만 최신 가이드라인을 모름 → 구식 치료 권고
- 가이드라인만 제공: 일반 원칙은 있지만 이 환자의 구체적 상황을 모름 → 교과서적 답변
- 논문만 제공: 연구 결과는 있지만 임상 적용 맥락이 없음 → 비실용적

의료 판단은 환자 데이터 + 근거(가이드라인/논문) + 임상 경험의 종합이다.

---

### 멀티소스 컨텍스트의 구조: 라벨링이 핵심

여러 출처를 통합할 때는 출처별 라벨링으로 AI가 정보의 성격을 구분하게 한다.

```
=== [환자 데이터] ===
55M. DM2+CKD3+HTN. Metformin 850mg bid, Losartan 100mg qd.
HbA1c 7.8, eGFR 38, K+ 4.9. NKDA.

=== [가이드라인: 대한당뇨병학회 2024] ===
CKD 동반 DM2에서 SGLT2i는 eGFR ≥ 20에서 시작 가능.

=== [논문: EMPA-KIDNEY, NEJM 2023] ===
Empagliflozin이 CKD 진행을 28% 감소 (HR 0.72).

=== [질문] ===
이 환자에게 SGLT2i 추가를 고려하고 있습니다.
```

라벨이 없으면 AI가 환자 데이터와 가이드라인의 일반론을 혼동할 위험이 있다.

---

### 출처 간 충돌 처리

여러 출처를 통합하면 필연적으로 충돌이 발생한다.

- 가이드라인 간: 한국은 eGFR < 30에서 Metformin 중단, ADA는 감량 후 유지 가능
- 가이드라인 vs 논문: 가이드라인은 eGFR ≥ 25에서 SGLT2i 시작, 최신 논문은 eGFR 20~25에서도 안전성 확인
- 일반 원칙 vs 환자 특이사항: DM2 1차 약제는 Metformin이지만 이 환자는 GI 부작용으로 감량 중

충돌을 AI가 무시하면 안 된다. 시스템 프롬프트에 충돌 처리 규칙을 명시한다.

```
여러 출처의 정보가 충돌할 경우:
1. 충돌 내용을 명시적으로 언급하세요
2. 한국 가이드라인을 1차 기준으로 삼으세요
3. 최신 근거가 다를 경우 양쪽을 모두 제시하세요
4. 환자 특이사항은 일반 원칙보다 우선합니다
```

---

### 라벨링 패턴

대괄호 라벨: `[환자데이터]`, `[가이드라인]`, `[논문]` — 간결하고 범용적이다.

메타데이터 포함: `[가이드라인 | 대한당뇨병학회 | 2024 | 신뢰도: 높음]` — AI가 정보의 최신성과 신뢰도를 고려하여 답변할 수 있다.

---

### 실전: 복합 만성질환의 다학제 자문 시나리오

환자: 62세 여성, DM2 + CKD4(eGFR 22) + HFrEF(EF 35%)

```
=== [환자 데이터] ===
62F. 약물: Empagliflozin 10mg, Sacubitril/Valsartan 100mg bid,
Bisoprolol 5mg, Furosemide 40mg. K+ 5.3, BNP 450.

=== [심장내과 관점] ===
HFrEF에서 MRA는 4대 필수약제. 단 K+ > 5.0이면 주의.

=== [신장내과 관점] ===
CKD4 + K+ 5.3 + ARNI 사용 중 → MRA 추가 시 고칼륨혈증 위험 높음.

=== [논문: FIDELIO-DKD] ===
Finerenone(비스테로이드 MRA)은 고칼륨혈증 위험이 낮으면서
CKD+DM 환자에서 신장·심혈관 보호 효과 확인.

=== [질문] ===
이 환자에게 MRA 추가가 필요한가? 어떤 약제가 적절한가?
```

이 컨텍스트를 받은 AI는 각 전문과의 관점을 통합할 수 있다. 심장내과 관점에서 MRA는 권고되지만, 신장내과 관점에서 K+ 5.3이므로 기존 Spironolactone은 위험하고, 최신 논문에 따르면 Finerenone이 대안이 될 수 있다. 단일 출처로는 불가능한 다학제적 종합 판단이 가능해진다.

---

### 멀티소스 통합 시 주의사항

1. 신뢰도 계층 명시: AI가 블로그 글과 RCT 논문을 동등하게 취급하면 안 된다
2. 최신성 표시: 2020년과 2024년 가이드라인이 충돌하면 최신 것을 우선한다
3. 환자 데이터 최우선: 일반 원칙이 아무리 강력해도, 이 환자의 K+ 5.3이 처방을 결정한다
4. 토큰 예산 관리: 모든 출처의 전문을 넣으면 윈도우가 넘친다. 관련 섹션만 발췌한다

## 핵심 정리
- 의료 AI는 EMR, 가이드라인, 논문 등 여러 출처를 동시에 참조해야 정확한 판단이 가능하다
- 출처별 라벨을 붙여 AI가 정보의 성격을 구분하게 한다
- 출처 간 충돌은 숨기지 말고 명시적으로 드러내며, 우선순위 규칙을 시스템 프롬프트에 포함한다
- 복합 만성질환의 다학제 자문처럼 여러 관점의 통합이 필요한 경우 멀티소스의 가치가 극대화된다

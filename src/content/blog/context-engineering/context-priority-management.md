## 한줄 요약
모든 정보를 컨텍스트에 넣을 수 없을 때, 환자 안전 > 임상 판단 > 편의 정보 순으로 우선순위를 매기고, 질문 유형에 따라 동적으로 컨텍스트를 재구성해야 한다.

## 본문

### 왜 우선순위가 필요한가

이상적으로는 모든 관련 정보를 제공하면 좋겠지만, 현실은 다르다.

- 공간 제약: 컨텍스트 윈도우는 유한하다
- 주의력 제약: LLM은 컨텍스트가 길어질수록 중간 부분을 놓치는 경향이 있다("Lost in the Middle")
- 비용 제약: 토큰 수에 비례하여 API 비용이 증가한다

따라서 무엇을 넣고 무엇을 뺄 것인가를 체계적으로 결정하는 프레임워크가 필요하다.

---

### 컨텍스트 우선순위 3단계 프레임워크

1단계: 환자 안전 정보 (반드시 포함)
- 알레르기 및 약물 과민 반응
- 현재 복용 중인 약물 (상호작용 확인용)
- 핵심 검사 수치 (K+, Cr, eGFR, 간수치)
- 금기 사항, 이전 부작용 이력

이 정보가 빠지면 AI가 위험한 처방을 권고할 수 있다. 어떤 상황에서도 생략하지 않는다.

2단계: 임상 판단 정보 (가능하면 포함)
- 현재 진단명과 병기
- 최근 검사 추이, 관련 가이드라인의 핵심 권고
- 치료 경과 및 이전 치료 반응

이 정보가 있으면 AI의 답변이 일반론에서 환자 맞춤형으로 바뀐다.

3단계: 배경/편의 정보 (공간이 남으면 포함)
- 사회력, 가족력, 상세 병력 서술
- 관련 논문 세부 내용, 이전 대화 중간 과정

---

### 동적 우선순위: 질문 유형에 따라 다른 컨텍스트

같은 환자라도 질문이 다르면 필요한 컨텍스트가 다르다.

"이 환자에게 Amlodipine을 추가해도 될까?"
→ 높은 우선순위: 현재 약물 목록, 알레르기, 신기능, 현재 혈압
→ 낮은 우선순위: 당뇨 검사 결과, 사회력

"이 환자의 HbA1c 관리 전략을 제안해 주세요"
→ 높은 우선순위: HbA1c 추이, 당뇨 약물, 신기능, 당뇨 가이드라인
→ 낮은 우선순위: 혈압 관련 세부 데이터

동적 컨텍스트 선택은 규칙 기반(키워드 분석으로 컨텍스트 선택), AI 기반(경량 모델이 질문 분류 후 컨텍스트 선택), 또는 사용자 명시("약물 상호작용 관점에서 봐주세요")로 구현할 수 있다.

---

### 실전: 외래 vs 응급 vs 입원 시 우선순위 차이

외래 진료 — 시간 충분, 만성 질환 관리 초점
```
1. 핵심 검사 추이 (3~6개월)
2. 현재 약물 및 순응도
3. 가이드라인 권고사항
4. 생활습관 정보
```

응급 상황 — 시간 촉박, 즉각적 위험 평가 초점
```
1. 알레르기 (즉시 필요)
2. 현재 약물 (상호작용)
3. 활력징후 + 응급 검사
4. 기저질환 목록
※ 장기 추이, 사회력은 후순위
```

입원 환자 — 반복 접근 가능, 종합 평가 초점
```
1. 입원 사유 및 현재 상태
2. 전체 약물 목록
3. 일별 검사 추이
4. 타과 자문 내용 + 가이드라인
```

---

### 우선순위 관리의 실수와 교훈

실수 1: 안전 정보 생략 — "공간 부족하니 알레르기는 빼자" → Penicillin 알레르기 환자에게 Amoxicillin 권고

실수 2: 모든 정보 투입 — EMR 전체를 붙여넣기 → K+ 5.8이 긴 텍스트 중간에 묻혀 AI가 놓침

실수 3: 고정 컨텍스트만 사용 — 모든 질문에 동일한 컨텍스트 → 약물 질문에 불필요한 가족력이 포함되고, 필요한 상호작용 DB 결과가 빠짐

교훈: 우선순위는 한 번 정하고 끝이 아니라, 질문과 상황에 따라 동적으로 재평가해야 한다.

## 핵심 정리
- 컨텍스트 우선순위는 환자 안전(필수) > 임상 판단(권장) > 배경 정보(선택) 3단계로 나눈다
- 같은 환자라도 질문 유형(처방, 검사 해석, 종합 평가)에 따라 필요한 컨텍스트가 다르다
- 외래·응급·입원 등 진료 환경에 따라 우선순위 체계가 달라져야 한다
- 알레르기, 금기, 핵심 수치 등 안전 정보는 어떤 상황에서도 생략하지 않는다

## 한줄 요약
의료 AI 컨텍스트는 가이드라인 업데이트 주기에 맞춰 관리되어야 하며, 날짜 태깅과 유효기간 설정, 변경 이력 추적 체계가 없으면 구버전 정보로 잘못된 답변을 생성할 위험이 있다.

## 본문

### 의료 가이드라인은 살아있다

의료 가이드라인은 고정된 문서가 아니다. 새로운 임상시험 결과, 약물 승인, 안전성 데이터에 따라 지속적으로 업데이트된다.

몇 가지 사례를 보자.

- 고혈압 진단 기준: 2017년 ACC/AHA 가이드라인에서 140/90에서 130/80 mmHg로 변경
- 당뇨 1차 약제: GLP-1 수용체 작용제의 심혈관 보호 효과 입증 후 권고 순위 상향
- COVID-19 치료: 수개월 단위로 권고안이 바뀜
- 항응고제 선택: DOAC이 와파린 대비 우위를 입증하면서 1차 선택 변경

AI 컨텍스트에 2022년 가이드라인이 그대로 남아 있으면, AI는 2022년 기준으로 답변한다. 사용자는 "AI가 이렇게 답했으니 맞겠지"라고 생각한다. 이것이 구버전 컨텍스트의 위험이다.

### 구버전 가이드라인이 남아있을 때의 위험

위험 1: 오래된 기준으로 치료 권고

```
[구버전 컨텍스트]
고혈압 1차 약제: thiazide 이뇨제 또는 CCB
(2018년 가이드라인 기준)

[현행 가이드라인]
심부전 동반 고혈압: SGLT2 억제제 추가 권고
(2023년 업데이트)
```

SGLT2 억제제의 심부전 보호 효과가 입증된 이후 가이드라인이 바뀌었지만, 구버전 컨텍스트에는 이 정보가 없다.

위험 2: 중단된 약물을 여전히 권고

안전성 문제로 시장에서 철수된 약물이 구버전 컨텍스트에 남아 있을 수 있다. AI는 이를 여전히 유효한 선택지로 제시할 수 있다.

위험 3: 진단 기준 불일치

진단 기준이 바뀌었는데 컨텍스트에 구기준이 남아 있으면, 진단 자체가 달라질 수 있다. 예를 들어 임신성 당뇨의 진단 기준은 IADPSG 기준 도입 후 변경되었다.

### 컨텍스트 버전 관리 체계

#### 1. 날짜 태깅

모든 컨텍스트 문서에 출처 날짜와 적용 버전을 명시한다.

```
[컨텍스트 메타데이터 예시]
---
문서명: 고혈압 진료 가이드라인 요약
출처: 대한고혈압학회 진료지침 2025
버전: 2025-v1
생성일: 2025-04-01
마지막 검토일: 2025-04-01
유효기한: 2026-03-31
상태: 활성
---
```

#### 2. 유효기간 설정

각 컨텍스트 문서에 유효기간을 설정하고, 만료 시 자동으로 리뷰 플래그를 올린다.

- 가이드라인 요약: 12개월 (연간 업데이트 주기에 맞춤)
- 약물 정보: 6개월 (신약 승인, 안전성 경고 주기)
- 진단 기준: 12개월
- 수가/보험 정보: 6개월 (보험 정책 변경 주기)

유효기간이 지났다고 무조건 삭제하는 것이 아니라, 리뷰 후 갱신 또는 폐기를 결정한다.

#### 3. 변경 이력 추적

Git과 같은 버전 관리 시스템을 활용해 컨텍스트 문서의 변경 이력을 추적한다.

```
[변경 이력 예시]
2025-04-01 v1.0 초기 버전 (2025 가이드라인 기반)
2025-09-15 v1.1 SGLT2 억제제 권고 등급 업데이트
2026-02-01 v2.0 2026 가이드라인 반영 전면 개정
```

변경 이력이 있으면 "이전에는 어떤 기준이었는지"를 확인할 수 있고, 문제 발생 시 원인을 추적할 수 있다.

#### 4. 다중 버전 공존 관리

가이드라인이 바뀌는 과도기에는 구버전과 신버전이 공존해야 한다. 예를 들어, 2025년 12월에 새 가이드라인이 발표되었지만 실제 임상 현장에서는 2026년 3월부터 적용되는 경우.

```
[다중 버전 컨텍스트]
# 현행 기준 (2025-12-31까지)
고혈압 1차 약제: ACEi/ARB, CCB, thiazide 중 선택

# 신규 기준 (2026-01-01부터 적용)
고혈압 1차 약제: 동일 + 심부전 동반 시 SGLT2i 우선 고려

# 전환 안내
2026년 1월부터 새 기준이 적용됩니다.
전환 기간(~2026년 3월)에는 두 기준 모두 참고하세요.
```

### 실전: 고혈압 가이드라인 변경 시 업데이트 플로우

2024년 가이드라인에서 2025년 가이드라인으로 변경되는 상황을 예로 들어보자.

Step 1: 변경 사항 식별
새 가이드라인의 주요 변경 사항을 목록화한다.
- 목표 혈압 변경 여부
- 1차 약제 권고 변경 여부
- 새로운 약물군 추가 여부
- 특수 환자군 (CKD, 당뇨, 심부전) 권고 변경 여부

Step 2: 기존 컨텍스트 대조
기존 컨텍스트 문서에서 변경 사항에 해당하는 부분을 찾는다.

Step 3: 컨텍스트 업데이트
변경된 부분을 새 가이드라인 기준으로 수정한다. 이때 원본 가이드라인의 정확한 문구와 페이지를 참조한다.

Step 4: 버전 번호와 메타데이터 갱신
문서 버전, 출처, 유효기간을 업데이트한다.

Step 5: 검증 및 배포
업데이트된 컨텍스트로 테스트 질문을 돌려 답변이 새 가이드라인과 일치하는지 확인한 뒤 활성화한다. 구버전은 삭제하지 않고 아카이브로 보관한다 — 과거 시점의 AI 답변을 검증할 때 필요하기 때문이다.

유효기간 만료 알림, 가이드라인 발행 모니터링(RSS 피드), 변경 사항 초안(AI 활용)은 자동화할 수 있지만, 임상적 정확성 확인과 최종 승인은 반드시 사람이 해야 한다.

## 핵심 정리

1. 의료 가이드라인은 매년 업데이트되며, 컨텍스트도 같은 주기로 관리해야 한다
2. 구버전 컨텍스트는 오래된 기준으로 치료를 권고하는 직접적 위험이 있다
3. 날짜 태깅, 유효기간 설정, 변경 이력 추적이 버전 관리의 3대 요소다
4. 가이드라인 전환기에는 구버전과 신버전이 공존하는 과도기 관리가 필요하다
5. 유효기간 알림과 변경 사항 초안은 자동화할 수 있지만, 임상적 검증은 사람이 한다

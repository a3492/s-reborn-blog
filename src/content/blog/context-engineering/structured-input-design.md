## 한줄 요약
AI에게 정보를 전달할 때 **형식(구조)**이 내용만큼 중요하다 — 같은 정보라도 구조화 방식에 따라 AI의 이해도와 답변 정확도가 달라진다.

## 본문

### 자유 서술 vs 구조화 입력

대부분의 사용자는 AI에게 자유 서술로 정보를 전달한다.

```
55세 남자인데요, 고혈압이랑 당뇨 있고, 요즘 어지럽다 하시고,
지난달 혈압은 150/95 정도였고, 메트포르민이랑 로사르탄 먹고 있어요.
```

필요한 정보는 있지만 **핵심 데이터가 서술 속에 묻혀 있다**. 나이, 진단명, 약물, 검사 결과가 한 문단에 뒤섞여 있다. 구조화된 입력은 이 문제를 해결한다.

```
[환자 기본]: 55세 남성
[진단]: 고혈압, 제2형 당뇨
[현재 약물]: Metformin 500mg bid, Losartan 50mg qd
[최근 검사]: BP 150/95, Cr 1.8 (상승 추세)
[주소]: 어지러움
```

**동일한 정보지만 AI는 두 번째 형식을 훨씬 정확하게 파싱한다.**

---

### 형식별 장단점 비교

| 형식 | 장점 | 단점 | 적합 |
|------|------|------|------|
| 마크다운 | 사람이 읽기 쉬움, LLM 호환 | 중첩 구조 제한 | 대화형 입력, 시스템 프롬프트 |
| XML | 계층 표현 강력, Claude 호환 | 토큰 과다 소비 | 자동화 파이프라인 |
| JSON | API 연동 용이 | 사람이 읽기 불편 | Function calling |
| 표 | 비교·시계열에 강함 | 중첩 불가 | 검사 추이, 약물 비교 |

Claude는 XML을 특히 잘 인식하고, GPT 계열은 마크다운과 JSON에 강하다. 상황과 모델에 맞게 선택해야 한다.

---

### 의료 데이터를 구조화하는 핵심 패턴

**패턴 1: 라벨링** — 각 정보 블록에 `[환자 데이터]`, `[가이드라인]`, `[질문]` 같은 라벨을 붙여 AI가 정보의 종류를 즉시 파악하게 한다.

**패턴 2: 구분자** — 정보 블록 사이에 `---`, `===` 등 구분자를 넣어 경계를 명확히 한다.

**패턴 3: 계층 구조** — 복잡한 정보는 상위-하위 관계를 명시한다. XML이나 들여쓰기 마크다운이 적합하다.

**패턴 4: 시계열 정렬** — 검사 결과를 날짜순으로 정렬하면 AI가 추이를 파악할 수 있다.

```
[Cr 추이]
2026-01: 1.2 → 2026-02: 1.5 → 2026-03: 1.8 ← 상승 추세
```

---

### 실전 예시: SOAP 노트 입력 구조화

SOAP 노트는 의료에서 가장 널리 쓰이는 기록 형식이다. 이를 AI 입력으로 구조화하면 진료 보조에 바로 활용할 수 있다.

```markdown
## SOAP Note — 2026-03-30

### S (Subjective)
- 주소: "일주일 전부터 어지러워요"
- 양상: 체위 변경 시 악화, 아침에 심함
- 동반 증상: 두통(-), 이명(-), 구역(+)

### O (Objective)
- VS: BP 150/95, HR 78, BT 36.5
- Lab: Cr 1.8, K+ 4.9, eGFR 38, HbA1c 7.2
- 현재 약물: Losartan 50mg qd, Metformin 500mg bid

### A (Assessment)
- [AI에게 요청]: 위 데이터를 바탕으로 감별 진단 제시

### P (Plan)
- [AI에게 요청]: 추가 검사 및 약물 조절 제안
```

이 구조에서 AI는 주관적 호소, 객관적 데이터, 분석 요청, 계획 요청을 명확히 구분한다. 자유 서술 대비 **감별 진단의 구체성과 약물 권고의 근거 제시**가 눈에 띄게 향상된다.

---

### 구조화 입력 설계 시 주의사항

1. **과도한 구조화를 피하라** — 구조가 너무 복잡하면 오히려 토큰이 낭비되고 핵심이 흐려진다
2. **모델에 맞게 선택하라** — Claude는 XML, GPT 계열은 마크다운/JSON에 강하다
3. **일관성을 유지하라** — 한 시스템 안에서 형식을 통일해야 AI가 패턴을 학습한다
4. **사람도 읽을 수 있게 하라** — 의료 AI의 출력은 결국 의료진이 검토한다

## 핵심 정리
- 자유 서술은 정보가 묻힌다 — 구조화된 입력은 AI의 이해도를 높인다
- 마크다운(범용), XML(계층), JSON(자동화), 표(비교)를 상황에 맞게 선택한다
- SOAP 노트처럼 의료 현장의 기존 구조를 활용하면 도입이 쉽다
- 구조화의 핵심은 라벨링, 구분자, 계층, 일관성이다

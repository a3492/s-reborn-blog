## 한줄 요약
컨텍스트 엔지니어링은 AI에게 **무엇을 물을지**가 아니라 **어떤 정보를 제공할지**를 설계하는 것이다 — 같은 질문이라도 컨텍스트가 다르면 답이 완전히 달라진다.

## 본문

### 프롬프트 엔지니어링의 한계

프롬프트 엔지니어링은 "질문을 잘 하는 법"이다. 역할 설정, CoT, Few-shot 등 질문의 구조를 다듬는 데 집중한다. 이것만으로도 답변 품질이 크게 올라간다.

하지만 한계가 있다.

```
[좋은 프롬프트]
당신은 내과 전문의입니다.
55세 남성, 고혈압+CKD3 환자의 1차 약제를 추천해 주세요.
```

이 프롬프트가 아무리 잘 설계돼 있어도, AI는 **이 병원의 처방 관행**, **환자의 과거 약물 반응**, **최근 변경된 가이드라인**을 모른다. 프롬프트에 이 정보가 없으면 일반론적 답변만 나온다.

컨텍스트 엔지니어링은 이 문제를 해결한다.

---

### 컨텍스트 엔지니어링이란

> AI 시스템에 제공되는 **모든 배경 정보(컨텍스트)**를 전략적으로 선택·구조화·관리하는 것

프롬프트가 "한 번의 질문"이라면, 컨텍스트는 그 질문을 둘러싼 **전체 환경**이다.

| 구분 | 프롬프트 엔지니어링 | 컨텍스트 엔지니어링 |
|------|---------------------|---------------------|
| 초점 | 질문의 구조와 형식 | 제공되는 정보 전체 |
| 범위 | 단일 프롬프트 | 시스템 프롬프트 + 문서 + 메모리 + 도구 |
| 비유 | 문진을 잘 하는 법 | 진료실 환경 전체를 세팅하는 법 |
| 핵심 질문 | "어떻게 물어볼까?" | "AI에게 무엇을 보여줄까?" |

---

### 컨텍스트의 구성 요소

AI 시스템이 받는 컨텍스트는 여러 층으로 이뤄져 있다.

**1. 시스템 프롬프트** — AI의 정체성과 규칙
```
당신은 S-Reborn 클리닉의 임상 AI입니다. 한국 보험 기준으로...
```

**2. 참조 문서** — RAG로 가져온 외부 지식
```
[검색된 가이드라인]: 대한고혈압학회 2022 권고안 p.34...
```

**3. 환자 데이터** — EMR에서 추출한 현재 환자 정보
```
[환자]: 55세 남성, eGFR 42, 현재 Losartan 50mg 복용 중...
```

**4. 대화 이력** — 이전 턴에서 나눈 대화
```
[이전 대화 요약]: ARB에서 CCB로 변경 고려 중이었음...
```

**5. 도구 결과** — 외부 도구(검색, 계산기 등)의 출력
```
[약물 상호작용 DB 조회 결과]: Amlodipine + Simvastatin...
```

이 5가지를 어떻게 조합하느냐가 컨텍스트 엔지니어링의 핵심이다.

---

### 의료에서 컨텍스트 엔지니어링이 중요한 이유

의료는 컨텍스트가 가장 중요한 분야 중 하나다.

- **같은 증상**이라도 환자 배경(나이, 기저질환, 복용 약물)에 따라 판단이 완전히 다르다
- **같은 약물**이라도 신기능, 간기능, 상호작용에 따라 안전성이 달라진다
- **같은 가이드라인**이라도 한국과 미국의 적용이 다르다

AI에게 이 컨텍스트를 제대로 제공하지 않으면, 답변은 항상 교과서적 일반론에 그친다.

---

### 실전 예시: 컨텍스트 유무 비교

**컨텍스트 없이**
```
고혈압 환자에게 ACE inhibitor를 처방할 때 주의사항은?
```
→ 일반론: 기침 부작용, 혈관부종, 임산부 금기 등

**풍부한 컨텍스트와 함께**
```
[시스템]: 당신은 S클리닉 임상 AI입니다. 한국 보험 기준.
[환자 데이터]: 55세 남성, eGFR 42, K+ 5.2, 현재 Losartan 50mg + Spironolactone 25mg
[가이드라인]: 대한신장학회 CKD 관리 권고안 2024
[질문]: 이 환자에게 ACE inhibitor 추가가 적절한가요?
```
→ 구체적: "eGFR 42 + K+ 5.2 + Spironolactone 병용 상황에서 ACEi 추가는 고칼륨혈증 위험. K+ 5.5 이상 시 금기. 현 상태에서 추가보다 ARB 용량 조절 우선 권고."

**같은 질문이지만 컨텍스트가 답의 질을 결정한다.**

---

### 이 시리즈에서 다루는 내용

컨텍스트 엔지니어링은 넓은 분야다. 이 시리즈에서는 다음을 단계별로 다룬다:

1. **개념** — 컨텍스트의 정의, 프롬프트와의 차이, 윈도우 전략
2. **설계** — 구조화된 입력, RAG 연결, 메모리 관리, 도구 통합
3. **의료 적용** — EMR 데이터 활용, 가이드라인 주입, 멀티소스 통합
4. **고급** — 컨텍스트 압축, 우선순위 관리, 평가 방법

## 핵심 정리
- 프롬프트 엔지니어링 = 질문을 잘 하는 법
- 컨텍스트 엔지니어링 = AI에게 제공하는 정보 환경 전체를 설계하는 법
- 의료에서는 환자 데이터 + 가이드라인 + 도구 결과를 AI에게 어떻게 제공하느냐가 답변 품질을 결정한다

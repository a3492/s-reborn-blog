## 한줄 요약
EMR 원본 데이터는 **비정형·중복·노이즈 투성이**라서 그대로 AI에게 넣으면 효과가 떨어진다 — 추출, 정리, 구조화 3단계 파이프라인과 비식별화를 거쳐야 한다.

## 본문

### EMR 원본 데이터의 문제

EMR에는 환자에 대한 방대한 정보가 있다. 하지만 그대로 AI에게 넣으면 여러 문제가 발생한다.

**비정형 텍스트**: 의사마다 기록 방식이 다르다.

```
의사 A: "HTN, DM2. BP 150/95. Losartan 50mg qd."
의사 B: "고혈압이 있는 환자분으로, 당뇨도 관리 중입니다. 혈압 150/95."
의사 C: "Dx) 1. Essential HTN  2. T2DM / BP) 150/95"
```

**중복 데이터**: 매 방문마다 동일한 진단명, 약물 목록이 반복된다. 3년 치 EMR에 "HTN"이 수십 번 나온다.

**노이즈**: 보험 청구용 코멘트, 행정 메모, 시스템 로그 등 임상적으로 무의미한 데이터가 섞여 있다.

---

### 3단계 파이프라인: 추출 → 정리 → 구조화

#### 1단계: 추출 — 어떤 필드를 가져올 것인가

**필수 추출**: 나이/성별, 활성 진단(ICD 포함), 현재 약물, 알레르기, 핵심 검사 결과

**선택 추출**: 활력징후 추이, 과거력, 가족력/사회력, 최근 1~2회 진료 노트

**추출 제외**: 보험 청구 코멘트, 행정 메모, 시스템 로그, 동의서 기록

#### 2단계: 정리 — 중복 제거와 표준화

```
[정리 전]
진단: HTN (2020-01), 고혈압 (2020-03), Essential hypertension (2021-06)
약물: Losartan 50mg 1T qd, 로사르탄 50밀리그램 1일 1회

[정리 후]
진단: HTN (2020-01~현재)
약물: Losartan 50mg qd
```

표준화 규칙: 진단명은 영문 약어 통일(HTN, DM2), 약물명은 성분명 영문 통일, 검사 수치는 단위 통일, 날짜는 YYYY-MM 형식.

#### 3단계: 구조화 — AI가 파싱하기 쉬운 형식

```
=== [환자 기본] ===
55세 남성
=== [활성 진단] ===
1. HTN (2020~)  2. DM2 (2024~)  3. CKD3b (2025~)
=== [현재 약물] ===
| 약물 | 용량 | 용법 |
|------|------|------|
| Losartan | 100mg | qd |
| Metformin | 850mg | bid |
=== [알레르기] ===
NKDA
=== [최근 검사 (2026-03)] ===
| 항목 | 결과 | 추이 |
|------|------|------|
| Cr | 1.8 | ↑ (1.2→1.5→1.8) |
| eGFR | 38 | ↓ |
| K+ | 5.1 | ↑ |
```

---

### 시계열 데이터의 표현

의료 데이터의 핵심 특성 중 하나는 **시간에 따른 변화(추이)**다. 단일 시점의 값보다 추이가 더 중요한 경우가 많다.

```
[비효과적] Cr: 1.8
[효과적] Cr 추이: 1.2 (2025-09) → 1.5 (2025-12) → 1.8 (2026-03) ↑
```

AI에게 추이를 제공하면 "Cr이 높다"가 아니라 "Cr이 상승 중이다"라는 동적 판단이 가능해진다.

---

### 자동화 vs 수동 변환

**자동화**: EMR API → JSON 추출 → 정리 스크립트 → 구조화 템플릿. 일관성과 속도가 장점이지만 초기 개발 비용이 든다.

**수동**: 의료진이 직접 정리하여 입력. 유연하지만 시간이 걸리고 일관성이 떨어진다.

현실적으로는 **자동 추출 + 의료진 검토·보완**의 하이브리드가 가장 효과적이다.

---

### 개인정보 비식별화 필수 사항

EMR 데이터를 AI에게 전달할 때 **비식별화는 선택이 아니라 의무**다.

**반드시 제거/변환**:
- 환자 이름 → "환자 A"
- 주민등록번호 → 나이/성별로 대체
- 연락처, 주소 → 제거
- 담당의 이름 → "담당의"

**유지해야 할 정보**: 나이, 성별, 진단명, 약물, 검사 수치, 날짜의 상대적 관계

**법적 근거**: 개인정보보호법 제23조(민감정보 처리 제한), 의료법 제21조(기록 열람). 외부 AI 서비스에 환자 정보를 직접 입력하는 것은 비식별화 후에만 허용된다.

---

### 변환 품질 검증 체크리스트

```
□ 핵심 진단이 모두 포함되었는가
□ 현재 약물이 정확한가 (중단된 약물이 포함되지 않았는가)
□ 알레르기 정보가 포함되었는가
□ 검사 수치의 단위가 정확한가
□ 개인 식별 정보가 모두 제거되었는가
□ 구조가 일관되고 AI가 파싱할 수 있는 형식인가
```

## 핵심 정리
- EMR 원본은 비정형, 중복, 노이즈로 인해 그대로 AI에 넣으면 효과가 떨어진다
- 추출(필요 필드 선택) → 정리(중복 제거, 표준화) → 구조화(AI 파싱 가능 형식)의 3단계를 거친다
- 시계열 데이터는 최신 값뿐 아니라 추이를 포함해야 AI가 동적 판단을 할 수 있다
- 개인정보 비식별화는 법적 의무이며, 환자 이름·주민번호·연락처는 반드시 제거한다

## 한줄 요약

MD 파일은 마크다운(Markdown) 문법으로 쓴 텍스트 문서예요.  
눈으로 읽기 좋고, 프로그램도 다루기 쉬워서 GitHub·노션·옵시디언·AI 도구가 모두 이 형식을 사랑합니다.

---

## 왜 알아야 할까?

🎯 “프롬프트 잘 쓰는 법”만 공부하다 보면, 어느 순간 이런 파일을 마주칩니다.

- 병원 내부 가이드를 노트 앱에 정리했다 →보내기 형식이 `.md`
- AI에게 RAG로 넣을 자료를 만든다 → 개발자가 “마크다운으로 쪼개 주세요”라고 한다
- 오픈소스 의료 도구 README를 본다 → 전부 마크다운

💡 MD를 알면 “그냥 메모장 파일”이 아니라, 디지털 세계와 AI가 이해하는 문서로 보이기 시작해요.

---

## MD 파일, 정체가 뭔가요?

파일 이름 끝이 `.md` 또는 `.markdown`이면 마크다운 파일입니다.  
본질은 평문(plain text)이에요. 워드처럼 숨겨진 서식 바이너리가 아니라, 글자 그대로 저장됩니다.

대신 제목·강조·목록만 아주 짧은 기호로 표시해요.

```markdown
## 회진 체크리스트
- 활력징후
- *신규* 검사 결과
```

이걸 앱이나 웹이 읽으면 예쁜 제목·불릿으로 렌더링해 줍니다.  
사람이 읽어도 되고, 기계가 읽어도 된다는 게 핵심이에요.

---

## 탄생 배경 — “웹은 멋진데, 글 쓰기는 너무 귀찮다”

2004년, 존 그루버(John Gruber)가 아론 스워츠(Aaron Swartz)의 도움을 받아 마크다운을 공개했습니다.  
당시 블로그·위키가 늘어나면서 HTML을 매번 타이핑하는 건 부담이었어요.

- `<h2>제목</h2>` 대신 `## 제목` 한 줄
- `<strong>강조</strong>` 대신 마크다운에서는 별표 두 개로 감싸 강조합니다 (예시는 위 코드 블록).

“읽기 쉬운 입력 → 그대로 HTML로도 바꿀 수 있게”가 목표였습니다.  
그래서 마크다운의 철학은 한 문장으로 말하면 “작성자의 뇌와 눈을 존중한다”에 가깝습니다.

---

## 마크업(Markup)과 마크다운(Markdown), 헷갈리면 이렇게

많은 분이 “마크업이랑 마크다운이 뭐가 달라요?”라고 물어요.

| 구분 | 뜻 |
|------|-----|
| 마크업(Markup) | 문서에 “이건 제목, 이건 목록”이라고 표시(태그)를 붙이는 방식 전체를 가리키는 넓은 말이에요. HTML, XML, LaTeX도 다 마크업 계열입니다. |
| 마크다운(Markdown) | 그중에서도 문법을 최소한으로 줄인 ‘가벼운’ 마크업 언어 하나의 이름이에요. |

비유하자면, “조리법”(마크업) 안에 “5분 레시피”(마크다운)가 있는 느낌이에요.  
HTML은 레스토랑 주방 매뉴얼이고, 마크다운은 집에서 바로 따라 쓰는 카드 한 장에 가깝습니다.

---

## JSON, XML, YAML과 나란히 놓으면 — AI 시대에 MD가 빛나는 이유

세 형식은 모두 구조화된 데이터를 잘 표현합니다. 그런데 역할이 달라요.

### JSON

- API·설정·로그에 최적. 기계가 파싱하기엔 최강이에요.
- 사람이 긴 JSON을 읽다 보면 괄호와 쉼표에 지칩니다.

### XML

- 스키마·검증·엔터프라이즈 연동에 강했던 시대의 표준.
- 태그가 길고 장황해서 짧은 메모·글쓰기에는 과해요.

### YAML

- 들여쓰기로 구조를 표현, 설정 파일(Kubernetes, CI)에 많이 씁니다.
- 공백 하나가 문법 오류로 이어질 수 있어, “긴 설명문” 본문에는 덜 씁니다.

### 마크다운(MD)

- 본문이 사람의 언어인 채로 남습니다. 제목·목록·코드 블록만 얇게 얹어요.
- Git diff로 무엇이 바뀌었는지 한눈에 보이고, 버전 관리와 찰떡입니다.
- LLM은 원래 자연어에 강합니다. 글이 길어도 문맥이 끊기지 않게 넣기 좋아요.
- RAG·문서 검색에서 청킹(잘라 넣기) 할 때, 마크다운의 `##` 경계가 의미 단위로 잘 맞는 경우가 많습니다.

정리하면, JSON/XML/YAML은 “계약서와 표”에 가깝고, MD는 “논문 초안과 메모”에 가깝습니다.  
AI 시대에는 둘 다 필요하지만, 지식을 쌓고 고치고 공유하는 층에서는 MD가 특히 많이 쓰입니다.

---

## 의료·연구 맥락에서 한 줄 팁

- 환자 식별 정보가 들어간 MD는 그대로 클라우드에 올리지 말고, 비식별화·내부 규정을 먼저 확인하세요.
- “AI에게 맡길 문서”는 표와 bullet을 마크다운으로 정리해 두면, 모델이 구조를 따라가기 쉬워요.

---

## PPT·발표 AI와 마크다운 — 왜 “먼저 MD로 쓰는” 사람이 유리할까?

슬라이드 도구(Gamma, Canva, Google Slides, PowerPoint Copilot 등)에 바로 잡담처럼 시키면 구조가 들쭉날쭉해지기 쉬워요.  
그래서 실무에서는 이런 2단계가 자주 쓰입니다.

1. 마크다운으로 뼈대 — 제목(`##`)마다 한 슬라이드(또는 한 섹션), bullet은 말할 포인트만.
2. AI 또는 슬라이드 앱에 붙여 넣기 — “이 구조 그대로 슬라이드로 바꿔줘. 한 슬라이드에 글자는 N줄 이내”처럼 형식을 고정합니다.

### 이렇게 쓰면 좋은 이유 (한 컵 커피 타임)

- 슬라이드는 ‘결과물’, 마크다운은 ‘검토하기 쉬운 초안’이에요. 교수·팀원에게 diff로 수정 내역을 보여 주기도 쉽습니다.
- 발표 직전에 문장만 고칠 때, PPTX 안을 뒤지지 않고 텍스트만 고쳐서 다시 생성하는 루틴이 가능해요.
- 개발자라면 Marp, Slidev처럼 “마크다운 → PDF/HTML 슬라이드”로 가는 파이프라인도 있어요. (병원 IT 정책·보안에 맞는 도구만 쓰세요.)

### 미니 예시: 학회 10분 발표 뼈대

```markdown
## 배경 — 왜 이 환자군인가
- inclusion 3줄

## 방법
- 후향적 / n=?

## 결과 — 한 장으로 말하면
- 주요 지표 2개만

## 한계 & 임상적 함의
```

이걸 AI에게 주면서 “각 ## 를 한 슬라이드로. 도표는 플레이스홀더만”이라고 붙이면, 밤새 PPT 지옥이 조금 짧아집니다.

---

## 재미있는 실제 활용 사례 모음 (버라이어티 팩)

같은 `.md`인데, 쓰는 사람에 따라 완전히 다른 무대가 돼요.

| 어디서 | 무슨 일 | 왜 MD인가 |
|--------|---------|-----------|
| 과·팀 단톡방 | “내일 교육 자료 공유해줘” 할 때 | 워드 파일 왔다 갔다보다 GitHub·노션보내기가 가벼움 |
| 전공의 | 케이스 회고를 `## 증상 / ## 검사 / ## 교훈`으로 | 나중에 저널 클럽이나 슬라이드로 재가공하기 쉬움 |
| 연구실 | IRB·분석 계획의 ‘사람이 읽는 버전’ | 표·체크리스트를 bullet로 두면 PI가 스크롤만 해도 됨 |
| 오픈소스·의료 AI | README, CONTRIBUTING | 전 세계가 같은 형식으로 기여 → LLM도 학습 데이터에 익숙 |
| Cursor·Copilot 규칙 | `AGENTS.md`, 프로젝트 가이드 | 코드와 같은 저장소에 두고 버전 관리 |
| 챗봇·에이전트 | 시스템 프롬프트 초안을 MD로 보관 | 버전 `v1`, `v2`를 파일명이나 폴더로 나누기 좋음 |
| 뉴스레터·블로그 | 제목·요약·본문 구조를 MD로 쓴 뒤 CMS에 붙여 넣기 | 한 번 쓴 글이 웹·이메일·슬라이드로 갈라지기 쉬움 |
| 강의 준비 | 한 학기치 강의를 `주차별.md`로 쌓기 | 내년엔 전년 파일 복사 후 diff만 보면 됨 |

하나만 기억해도 됩니다. “나중에 다른 형태로 바꿀 가능성이 있는 글”은 MD에 가깝게 두는 편이 이긴다는 것.

## 오늘의 과제 — 마크다운 근육 15분 루틴

아래 중 하나만 해도 통과예요. (전부 하면 보너스 없음… 아니, 보너스는 내일 발표가 덜 괴로움입니다.)

### 과제 A — “내 전용 템플릿” 만들기

1. 빈 파일을 `my-slide-bone.md` 같은 이름으로 저장합니다.
2. `##` 세 개 이상으로 다음 주에 말할 일(회진, 학회, 팀 회의 아무거나)의 목차만 적습니다.
3. 각 `##` 아래 bullet 최대 3개만 적습니다.

### 과제 B — AI와 한 판 (구조 고정)

방금 파일을 AI에게 주고 이렇게 말해 보세요.

> “이 마크다운의 `##`마다 슬라이드 하나로 바꿔줘. 한 슬라이드 bullet은 5개 이하. 환자 식별 정보는 넣지 마.”

나온 결과에서 한 슬라이드만 직접 손으로 고쳐 보세요. 어디가 불편했는지 메모해 두면 다음 프롬프트가 진화합니다.

### 과제 C — 1분 복습

아래를 눈 감고 말해 보세요.

- `#`와 `##`의 차이
- 별표 두 개로 굵게 표시 한 번 쓰기
- `-` 로 bullet 세 줄

막히면 위 “MD 파일, 정체가 뭔가요?” 절로 돌아가면 됩니다.

## 다음 화 예고 — 옵시디언(Obsidian)과는 무슨 사이?

마크다운을 이해했다면, 다음 질문은 자연스럽게 이어집니다.

> “그럼 요즘 의사·연구자들이 말하는 옵시디언은 뭐예요?”

옵시디언은 마크다운 파일을 폴더에 쌓아 두는 방식으로 “두 번째 뇌”를 만드는 앱이에요.  
위키 링크 `[[이렇게]]`, 그래프 뷰, 태그, 플러그인으로 지식이 서로 연결됩니다.

이어서 읽기 → [옵시디언이 뭐예요? — 마크다운 폴더를 ‘두 번째 뇌’로 바꾸는 법](/blog/aerini/obsidian-and-markdown/) 에서 옵시디언이 왜 마크다운을 기본으로 쓰는지, 위키 링크·그래프, 워드·노션과의 차이, AI 시대에 같이 언급되는 이유를 풀어드렸어요.

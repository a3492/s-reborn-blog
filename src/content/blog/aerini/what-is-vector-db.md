## 한줄 요약

Vector Database는 텍스트를 숫자 벡터로 저장해서 AI가 '의미가 비슷한 것'을 빠르게 찾을 수 있게 하는 데이터베이스예요. RAG의 핵심 부품이에요.

---

## 왜 알아야 할까?

- 💡 AI가 병원 내부 문서를 검색하는 원리를 이해할 수 있어요
- 💡 RAG 시스템이 어떻게 작동하는지 알게 돼요
- 💡 의료 지식 베이스를 AI가 활용하는 방법을 이해해요
- 💡 병원 내 AI 도구 도입을 논의할 때 기술적 대화가 가능해져요

## 이게 뭐예요?

**Vector Database(벡터 데이터베이스)**는 텍스트나 이미지를 숫자 벡터(Embedding)로 변환해서 저장하는 특수한 데이터베이스예요. 일반 데이터베이스가 키워드 일치로 검색한다면, 벡터 DB는 의미적 유사도로 검색해요.

예를 들어 "고혈압 치료 프로토콜"을 검색하면 "HTN management guideline"도 찾아줘요. 같은 의미를 가진 다른 표현도 인식하는 거예요.

🏥 **비유 1: 키워드 검색 vs 의미 검색**

병원 EMR에서 "흉통"을 키워드 검색하면 정확히 "흉통"이라고 쓰인 기록만 나와요. "가슴 답답함", "chest pain", "흉부 불편감"은 안 나와요. 하지만 Vector DB 기반 검색은 이 표현들을 모두 같은 의미로 인식해서 찾아줘요.

🏥 **비유 2: 고혈압 약 검색**

일반 검색: "고혈압 약" 검색 → "고혈압 약"이 정확히 있는 문서만 반환

Vector DB 검색: "고혈압 약" 검색 → "HTN medication", "항고혈압제", "혈압강하제" 포함 문서까지 반환

의료 전문 용어와 일반 용어, 영어와 한국어 혼재를 자연스럽게 처리해요.

**한줄 정리:** Vector DB = 의미를 이해하는 검색 엔진. 같은 뜻이면 다른 표현도 찾아줘요.

## 예시를 들어볼게

**예시 1 — 유사 환자 케이스 검색**

> 질문: "이 환자와 비슷한 증상 패턴의 과거 케이스 찾아줘"
> Vector DB: 증상 설명을 벡터로 변환 → 수천 건 케이스 중 가장 유사한 케이스 반환

정확한 진단명이 달라도 증상 패턴이 유사하면 찾아줘요.

**예시 2 — 의료 문헌 검색**

> 질문: "당뇨 환자의 수술 전 혈당 관리 지침"
> Vector DB: 관련 논문과 가이드라인을 의미 기반으로 검색 → "perioperative glycemic control in diabetics" 영문 문서도 반환

언어 장벽 없이 의미 기반으로 검색해요.

**예시 3 — 병원 프로토콜 검색**

> 의사: AI 챗봇에게 "패혈증 환자 초기 처치 어떻게 해?"
> AI가 병원 내부 프로토콜 Vector DB에서 관련 문서 검색 → 병원 표준 프로토콜을 기반으로 답변

AI가 외부 지식이 아닌 우리 병원 프로토콜을 참조해서 답해줘요.

## 의료에서는 이렇게 써요

**활용 1: RAG 기반 의료 AI 시스템**

❌ 일반 LLM만 사용: AI가 학습 시점 기준 지식만 사용 → 최신 가이드라인 반영 안 됨

✅ Vector DB + LLM (RAG): 최신 의료 가이드라인을 Vector DB에 넣고 실시간 검색 → 항상 최신 지식 기반 답변

RAG는 Vector DB가 있어야 작동해요.

**활용 2: 병원 내부 지식 AI화**

❌ 기존 방식: 병원 프로토콜을 찾으려면 인트라넷에서 직접 검색

✅ Vector DB 적용: 챗봇에 "우리 병원 DVT 예방 프로토콜이 뭐야?"라고 물으면 즉시 답변

병원 내부 문서를 AI가 실시간으로 참조해서 답변할 수 있어요.

**활용 3: 임상 의사결정 지원**

❌ 키워드 검색 시스템: "패혈증"으로 검색해야 패혈증 관련 문서가 나옴

✅ Vector DB 기반: "열이 나고 혈압이 떨어지는 환자"로 검색해도 패혈증 프로토콜을 찾아줌

의료진이 기억나는 표현 그대로 검색해도 관련 프로토콜이 나와요.

## 주의할 점 ⚠️

⚠️ **Vector DB도 최신 정보가 없으면 못 찾아요**

Vector DB에 넣지 않은 문서는 검색 결과에 나오지 않아요. 새 가이드라인이 나오면 Vector DB를 업데이트해야 해요.

⚠️ **검색 품질은 Embedding 모델에 달려 있어요**

Vector DB의 검색 능력은 어떤 Embedding 모델로 벡터를 만들었는지에 크게 의존해요. 의료 데이터에는 의료 특화 Embedding 모델을 쓰는 게 좋아요.

⚠️ **규모가 커지면 관리 비용도 커져요**

병원 전체 문서를 Vector DB로 만들면 저장 공간과 계산 비용이 상당해요. 자주 쓰이는 핵심 문서부터 시작하는 게 현실적이에요.

## 한줄 요약

Token은 AI가 문장을 쪼개서 읽는 가장 작은 단위입니다.
Token 개수 = AI 사용 비용이기 때문에 꼭 알아야 해요.

---

## 왜 알아야 할까?

🎯 ChatGPT를 병원에서 쓰다가 갑자기 "한도 초과" 메시지를 받은 적 있나요?

또는 긴 환자 기록을 입력했더니 갑자기 AI가 뒷부분을 무시하거나
엉뚱한 답을 한 적이 있나요?

💡 이게 다 Token 때문이에요.

Token을 이해하면:
- ✅ ChatGPT 사용 비용을 절반으로 줄일 수 있어요
- ✅ AI가 왜 긴 입력을 처리 못 하는지 이해해요
- ✅ 더 효율적인 프롬프트를 쓸 수 있어요
- ✅ 유료 플랜 결정에 도움이 돼요

---

## 이게 뭐예요?

### 정의
Token은 AI가 텍스트를 처리할 때 사용하는 가장 작은 단위입니다.

영어는 보통 단어가 1~2 Token,
한국어는 단어 하나가 2~4 Token 정도예요.

### 비유: 요리 재료 단위
레시피를 보면 "마늘 2쪽", "파 한 줄기" 처럼 나와 있어요.
요리 재료를 세는 단위가 있듯이,
AI도 텍스트를 세는 단위가 있어요 — 그게 Token이에요.

요리 비용이 재료량에 따라 다르듯,
AI 비용도 Token 수에 따라 달라요.

### Token이 돈이에요
ChatGPT API 기준 (2024):
- 입력 1,000 Token: 약 0.5원 (GPT-4o 기준)
- 출력 1,000 Token: 약 1.5원

의료 기록 1건 = 약 500~1,000 Token
환자 30명 기록 = 15,000~30,000 Token = 수십 원

하루, 한 달로 쌓이면 비용이 됩니다.

---

## 예시를 들어볼게

### 예시 1: 한국어 Token 수
"환자는 고혈압이 있습니다."

대략 10~15 Token

"55세 남성 환자, 2주간 두통 지속, 혈압 160/100 측정,
기존 항고혈압제 복용 중, 최근 스트레스 증가 호소."

대략 50~70 Token

### 예시 2: Token 한계
ChatGPT-4의 한 번에 처리 가능한 Token:
- 입력 + 출력 합쳐서 약 128,000 Token (최신 버전 기준)

긴 환자 기록 10건이 각 2,000 Token이라면
20,000 Token → 처리 가능

하지만 입력이 너무 길어지면 앞부분을 "잊어버리는" 현상이 생겨요.

### 예시 3: 비용 비교
❌ 비효율:
"오늘 외래 환자 전체 기록을 한 번에 붙여넣기" (5,000 Token)

✅ 효율:
"환자별로 핵심 정보만 요약해서 입력" (200 Token)

25배 차이!

## 의료에서는 이렇게 써요

### Token 절약 방법

방법 1: 핵심만 입력하기

❌ 전체 입력:
"환자 이름: 홍길동, 나이: 55세, 성별: 남성,
내원 날짜: 2026년 3월 31일, 주호소: 두통,
현재 복용 약: 암로디핀 5mg, 리시노프릴 10mg,
혈압 측정 결과: 160/100mmHg..."

✅ 핵심만 입력:
"55세 남성, 두통, BP 160/100, 고혈압약 복용 중.
감별 진단 요청."

방법 2: 시스템 프롬프트는 짧게

"당신은 전문 의사입니다. 간결하게 답해주세요."
→ 이것만으로도 충분해요.

긴 배경 설명은 Token만 낭비할 수 있어요.

방법 3: 답변 길이 지정

"100단어 이내로 답해줘" 라고 하면
출력 Token이 줄어서 비용이 절감돼요.

---

## 주의할 점 ⚠️

### 주의 1: 너무 짧아도 문제예요
Token을 아끼려고 너무 짧게 쓰면
AI가 맥락을 이해 못 해서 엉뚱한 답을 해요.

✅ 균형: 필요한 맥락은 주되, 불필요한 정보는 빼세요.

### 주의 2: 무료 버전은 Token 제한이 있어요
ChatGPT 무료 버전은 하루 사용량 제한이 있어요.
병원에서 많이 쓰려면 유료 플랜을 고려하세요.

### 주의 3: 모델마다 Token 계산이 달라요
GPT-4, Claude, Gemini 모두 Token 계산 방법이 달라요.
한국어는 특히 영어보다 Token을 더 많이 씁니다.

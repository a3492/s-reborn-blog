"딥러닝(Deep Learning)"이라는 이름은 누가 지었을까요? 그리고 그 이름을 만든 사람이 왜 나중에 AI를 두려워하게 됐을까요? 의학에서 항생제를 발견한 플레밍이 내성균을 걱정한 것처럼 말이죠.

## 딥러닝이라는 이름의 탄생

2006년, 캐나다 토론토대학교 교수 제프리 힌튼(Geoffrey Hinton)이 논문 "A Fast Learning Algorithm for Deep Belief Networks"를 발표했습니다. 여기서 "deep"이라는 단어를 강조했습니다.

### 왜 "deep"(깊다)인가?

| 구분 | 설명 |
|------|------|
| 얕은 신경망 (Shallow) | 입력층 + 출력층 = 1~2개 층 |
| 깊은 신경망 (Deep) | 은닉층이 여러 개 = 3층 이상 |
| 딥러닝의 핵심 | 층이 깊어질수록 더 복잡한 패턴 학습 가능 |

피부의 피상적인 관찰 → 조직 검사 → 세포 분석처럼, 딥러닝도 층이 깊어질수록 더 추상적인 특징을 파악합니다.

## 제프리 힌튼의 여정

| 연도 | 사건 |
|------|------|
| 1986 | 역전파 알고리즘 공동 발명 — 신경망 학습의 핵심 |
| 2006 | "Deep" 신경망 개념으로 AI 겨울 탈출 |
| 2012 | 제자들과 AlexNet — 이미지 인식 혁명 |
| 2013 | Google에 합류 (4,400만 달러에 스타트업 인수) |
| 2023 | Google 퇴사 — "AI 위험성을 자유롭게 말하기 위해" |
| 2024 | 노벨 물리학상 수상 |

## 딥러닝의 대부가 왜 AI를 두려워하는가

힌튼은 구글을 떠나며 이렇게 말했습니다. "평생 한 일을 후회하지는 않지만, AI가 인간보다 더 똑똑해지는 날을 걱정한다."

✅ 힌튼의 경고는 단순한 공포가 아닙니다. 기술의 내부를 가장 잘 아는 사람의 우려입니다.

⚠️ "딥러닝의 아버지"가 AI 위험을 경고한다는 건, 의사가 자신이 개발한 약의 부작용을 경고하는 것과 같습니다.

❌ AI 위험론을 무시하는 것도, 과도하게 두려워하는 것도 균형 잡힌 태도가 아닙니다.

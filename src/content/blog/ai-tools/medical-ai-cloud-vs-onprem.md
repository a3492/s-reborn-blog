## 한줄 요약
병원 규모와 규제 준수 요건에 따라 답이 다르다 — 의원은 클라우드, 대형병원은 하이브리드, 그리고 어디서든 한국 개인정보보호법 제28조의8은 피할 수 없다.

## 왜 지금 이 선택이 중요한가

AI 영상 판독, 챗봇 환자 응대, 임상 의사결정 지원 — 이 모든 서비스를 병원이 도입하려면 인프라 선택이 먼저다. 클라우드로 가면 빠르지만 규제 리스크가 있고, 온프레미스로 가면 안전하지만 초기 투자와 유지보수가 발목을 잡는다.

## 핵심 비교 테이블

| 항목 | 클라우드 | 온프레미스 | 하이브리드 |
|------|---------|-----------|----------|
| 초기 투자 | 낮음 (월정액) | 높음 (서버 5천만~수억) | 중간 |
| 운영 비용 | 사용량 비례, 예측 어려움 | 인건비+유지보수 고정 | 부분 고정 |
| 확장성 | 즉시 확장 가능 | 하드웨어 추가 필요 | 클라우드 쪽 탄력적 |
| 보안 | 공급자 의존 (SOC 2, ISO 27001) | 자체 통제 | 민감 데이터는 내부 |
| 구현 속도 | 수주~수개월 | 6개월~2년 | 6개월~1년 |
| 규제 준수 | 국외이전 위험 있음 | 국내 보존 확실 | 설계에 따라 다름 |
| 재해 복구 | 공급자가 처리 | 별도 DR 구축 필요 | 클라우드가 DR 역할 |

## 클라우드 옵션: 주요 서비스 비교

### AWS HealthLake
- **용도**: FHIR R4 기반 의료 데이터 저장/쿼리/ML 파이프라인
- **가격**: 저장 $0.023/GB/월, 쿼리 $0.01/1000건 (2026년 기준)
- **장점**: SageMaker와 네이티브 연동, HIPAA 적격 서비스
- **한계**: 서울 리전 데이터 보존 가능하나, 미국 본사 접근 가능성으로 국외이전 법적 쟁점 잔존

### Azure Health Data Services
- **용도**: FHIR, DICOM, MedTech(IoT) 통합 플랫폼
- **가격**: FHIR API $0.30/1000 요청, DICOM $0.14/GB/월
- **장점**: Epic, Cerner 등 주요 EMR 연동 커넥터 기본 제공
- **한계**: 한국 규정 준수 컨설팅 별도 필요

### Google Cloud Healthcare API
- **용도**: DICOM, FHIR, HL7v2 처리 + Vertex AI 연동
- **가격**: DICOM 저장 $0.023/GB/월, FHIR $0.01/1000 ops
- **장점**: DeepMind 연구 성과 연동, BigQuery와 임상 데이터 분석 용이
- **한계**: 구글 특유의 서비스 중단 이력, 한국 의료계 신뢰도 낮음

## 온프레미스 고려 사항

### 개인정보보호법 제28조의8 (국외이전 제한)
2024년 개정 개인정보보호법에 따라 **민감정보(건강정보 포함)의 국외이전은 정보주체 별도 동의 또는 보호위원회 허가**가 필요하다. 클라우드 해외 서버로 환자 데이터가 전송되면 위반 소지가 발생한다. 서울 리전 사용도 계약서상 미국 법인이 접근 가능한 구조라면 안전하지 않다.

### PACS 연동 구조
```
[영상 장비] → [PACS 서버 (온프레미스)] → [AI 분석 서버 (온프레미스)]
                                           ↓
                              [RIS/EMR] ← [소견 자동 기록]
```
온프레미스 AI는 PACS에서 DICOM 파일을 로컬 네트워크로 수신해 분석하므로 데이터가 병원 외부로 나가지 않는다.

### 초기 투자 현실
- GPU 서버 (NVIDIA A100 기준): 1대당 약 4천만~8천만 원
- PACS 연동 소프트웨어 라이선스: 연 1천만~5천만 원
- IT 인력 (전담 1명): 연 5천만~8천만 원
- 총 3년 TCO: 소형 시스템 기준 3억~5억 원

## 규모별 권장 구성

| 의료기관 규모 | 권장 방식 | 이유 |
|------------|---------|------|
| 의원 (1~3인) | 클라우드 SaaS | 초기 투자 불가, IT 인력 없음 |
| 중소병원 (30~200병상) | 클라우드 (서울 리전) + 법무 검토 | 비용 효율, 규제 대응은 계약으로 |
| 종합병원 (200~500병상) | 하이브리드 | 민감 데이터 내부, 분석은 클라우드 |
| 상급종합병원 (500병상+) | 온프레미스 + 프라이빗 클라우드 | 규제 확실, 대용량 데이터 처리 |

## 하이브리드 아키텍처 예시

```
[환자 데이터] → [온프레미스 PACS/EMR]
                        ↓ (익명화/탈식별화)
              [DMZ 게이트웨이]
                        ↓
              [클라우드 AI 분석]  ← 탈식별화 영상만 전송
                        ↓
              [소견 결과 반환]
                        ↓
              [온프레미스 EMR 기록]
```
민감 원본 데이터는 병원 내부에 두고, AI 연산만 클라우드에서 처리하는 구조다. 탈식별화 수준과 재식별 가능성을 법무팀이 사전 검토해야 한다.

## 핵심 정리
- 클라우드는 빠르고 저렴하지만, 국내 의료 규제상 국외이전 리스크를 계약으로 명확히 해야 한다
- 온프레미스는 규제 측면에서 가장 안전하나, 초기 3억~5억 원 투자와 전담 IT 인력이 필수다
- 대부분의 대형병원은 민감 원본 데이터는 내부, AI 연산은 클라우드로 처리하는 하이브리드를 선택한다
- 개인정보보호법 제28조의8 위반 시 과징금은 매출액의 3% 이하 — 법무 검토 없이 클라우드 도입하지 말 것
- 의원급은 국내 SaaS 업체(뷰노, 루닛 등)의 클라우드 서비스가 현실적 대안이다

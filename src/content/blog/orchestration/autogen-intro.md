## 한줄 요약
AutoGen 에이전트들은 대화로 협력한다. 슬랙에서 팀원들이 메시지를 주고받으며 일하는 것처럼.

## 본문

### Microsoft가 AutoGen을 만든 이유

2023년 9월, Microsoft Research가 AutoGen을 발표했다.

당시 LangChain이 시장을 주도하고 있었고, LangGraph는 아직 나오기 전이었다. Microsoft 팀의 문제의식은 달랐다: "에이전트들이 미리 정해진 그래프나 역할에 갇히지 않고, 사람처럼 자유롭게 대화하면서 문제를 풀 수 있다면 어떨까?"

사람들이 팀으로 일할 때를 생각해보면, 슬랙 채널에서 "이 부분은 내가 보기에 이상한데, 네가 확인해줄 수 있어?"처럼 자유롭게 소통한다. 어떤 순서로 누가 말할지 미리 정해두지 않는다.

AutoGen은 AI 에이전트들 사이에서 이것을 구현하려 했다.

---

### AutoGen의 핵심: 대화 기반 협력

LangGraph와 비교하면 차이가 명확하다.

**LangGraph 방식**:
```
[데이터 수집] → [분석] → {이상 징후?} → [에스컬레이션 / 정상 처리]
```
흐름이 미리 정의된 그래프로 고정된다.

**AutoGen 방식**:
```
내과 에이전트: "이 환자 복약 목록 보니 K+가 높을 수 있는 조합이 있어요."
약사 에이전트: "리시노프릴 + 스피로노락톤 조합이죠? 최근 K+ 수치 어떤가요?"
Lab 에이전트: "K+ = 5.3 mEq/L로 최근 측정됐습니다."
약사 에이전트: "정상 범위지만 추세 봐야 해요. 지난 달 비교가 필요합니다."
Lab 에이전트: "지난 달 5.0 → 이번 달 5.3. 상승 추세입니다."
약사 에이전트: "스피로노락톤 용량 감량 또는 전해질 모니터링 강화를 권고합니다."
```

대화 흐름이 사전에 정의되지 않는다. 각 에이전트가 이전 메시지를 보고 자신이 기여할 수 있는 내용을 말한다.

---

### AutoGen 구성 요소

**AssistantAgent** — LLM 기반 에이전트

역할과 전문성을 가지고 대화에 참여한다. 코드 작성, 분석, 권고 등을 담당한다.

**UserProxyAgent** — 사람 대리 에이전트

실제 사람의 역할을 한다. 코드 실행, 사람의 입력 요청, 대화 시작/종료 등을 담당한다. 자동 응답 모드로 설정하면 사람 없이도 작동한다.

**GroupChat** — 멀티 에이전트 대화방

여러 에이전트가 하나의 대화에 참여한다. GroupChatManager가 누가 다음에 발언할지 조율한다.

---

### AutoGen이 빛나는 케이스: 데이터 분석 협업

의료 데이터 분석처럼 코드 실행이 필요한 탐색적 작업에서 AutoGen이 강점을 보인다.

```
의사: "지난 6개월 당뇨 환자 HbA1c 추이를 분석해줘"

코딩 에이전트: "환자 데이터를 조회하고 그래프를 그리겠습니다."
              [Python 코드 작성 후 실행]
              "HbA1c 평균이 8.2 → 7.6으로 개선됐습니다.
               그래프를 보면 3개월째부터 목표치(7.0) 도달이 시작됐습니다."

통계 에이전트: "모집단을 살펴보면 개선 폭이 큰 그룹(>1.0 감소)과
               비개선 그룹(<0.3 감소)이 명확히 나뉩니다.
               두 그룹의 차이를 분석해볼 만합니다."

코딩 에이전트: "두 그룹을 나눠서 처방 유형, 복약 순응도 비교를 실행하겠습니다."
              [추가 분석 코드 실행]

해석 에이전트: "처방 유형보다 복약 순응도가 더 강한 예측 변수입니다.
               복약 교육 강화가 개선에 도움이 될 것 같습니다."
```

이 과정에서 어떤 분석을 할지 미리 정하지 않았다. 각 단계 결과를 보고 다음 분석이 자연스럽게 결정됐다.

---

### LangGraph vs CrewAI vs AutoGen 비교

| 항목 | LangGraph | CrewAI | AutoGen |
|---|---|---|---|
| 핵심 개념 | 그래프 흐름 | 역할 팀 | 대화 협력 |
| 흐름 예측 가능성 | 높음 | 중간 | 낮음 |
| 코드 실행 | 가능 | 제한적 | 강점 |
| 디버깅 편의성 | 높음 | 중간 | 낮음 |
| 적합한 케이스 | 정형화된 워크플로우 | 역할 기반 팀 | 탐색적 분석, 연구 |

---

### 의료 AI에서 AutoGen의 적용 범위

**적합한 케이스**

- 의료 데이터 탐색 분석 (어떤 분석이 필요한지 미리 모름)
- 임상 연구 보조 (연구자와 AI가 대화하며 분석 방향 결정)
- 의학 교육 (케이스 디스커션, 모의 컨퍼런스)

**부적합한 케이스**

- 정형화된 환자 처리 워크플로우 (예측 불가능성이 위험)
- 실시간 임상 의사결정 지원 (응답 일관성 중요)
- 규제 요구사항이 있는 시스템 (대화 흐름이 감사 추적에 어렵)

의료 환경에서 AutoGen은 "뒤에서 분석을 돕는 도구"로 쓰기 적합하다. 직접 환자 케어에 관여하는 시스템에는 예측 가능한 LangGraph가 더 안전하다.

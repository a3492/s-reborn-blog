## 한줄 요약
오케스트레이션은 여러 AI·도구·에이전트가 각자의 역할을 하면서 하나의 목표를 달성하도록 지휘하는 기술이다.

## 본문

### 왜 하나의 AI로는 부족한가

ChatGPT에게 "우리 병원 지난달 외래 환자 통계를 분석해서 PDF 보고서로 만들어줘"라고 하면 어떻게 될까?

모델은 할 수 없다고 한다. 이유는 세 가지다:
1. 병원 DB에 접근할 수 없다
2. 실시간 데이터가 없다
3. PDF를 직접 만들 수 없다

이걸 해결하려면 여러 전문 도구와 AI가 협력해야 한다:

```
DB 조회 도구 → 데이터 추출
통계 분석 AI → 패턴 발견
차트 생성 도구 → 시각화
요약 AI → 내러티브 작성
PDF 생성 도구 → 최종 출력
```

이 다섯 단계를 올바른 순서로, 앞 단계의 결과를 다음 단계에 넘기면서, 실패하면 재시도하고, 전체 진행을 모니터링하는 것 — 이게 **오케스트레이션**이다.

---

### 오케스트라 비유

오케스트라를 생각해보자. 바이올린, 첼로, 오보에, 타악기 — 각자 다른 악기를 연주하는 연주자들이 있다. 각 연주자는 자기 파트를 완벽하게 안다.

그런데 베토벤 교향곡을 연주하려면 지휘자가 필요하다. 지휘자는 직접 악기를 연주하지 않는다. 대신:
- 누가 언제 연주할지 결정한다
- 전체 흐름과 속도를 조율한다
- 앞 파트가 끝나야 다음 파트가 시작하도록 한다
- 실수가 생기면 조율한다

AI 오케스트레이션에서 지휘자는 **오케스트레이터(Orchestrator)**다. 연주자는 각 **에이전트(Agent)** 또는 **도구(Tool)**다.

---

### 오케스트레이션의 핵심 개념

**태스크 분해 (Task Decomposition)**
복잡한 목표를 더 작은 하위 태스크로 나눈다.

"환자 안전 개선 보고서 작성"
→ 낙상 사고 데이터 조회
→ 고위험 환자 식별
→ 유사 병원 벤치마크 조사
→ 개선 권고안 초안
→ 최종 보고서 구성

**의존성 관리 (Dependency Management)**
어떤 태스크가 다른 태스크의 결과를 필요로 하는지 파악한다. 병렬로 실행할 수 있는 것과 순차적으로 해야 하는 것을 구분한다.

**상태 관리 (State Management)**
여러 단계를 거치는 동안 중간 결과와 컨텍스트를 유지한다. 5단계 작업 중 3단계에서 실패하면 처음부터 다시 하는 게 아니라 3단계부터 재시도해야 한다.

**오류 복구 (Error Recovery)**
에이전트나 도구가 실패했을 때 어떻게 할 것인가. 재시도? 다른 방법으로 우회? 포기하고 부분 결과 반환?

---

### 오케스트레이션 vs 단순 파이프라인

단순 파이프라인:
```
A → B → C → D
```
순서가 고정되어 있다. A 실패하면 전체 실패. 분기 없음.

오케스트레이션:
```
        ┌→ B ─┐
A ──────┤     ├──→ D
        └→ C ─┘
        (B, C 병렬)
        실패 시 E로 우회
```
동적 분기, 병렬 실행, 실패 처리, 조건부 흐름이 있다.

### 의료 현장 오케스트레이션 예시

**임상 의사결정 지원 시스템**

```
입력: 환자 ID + 의사의 질문 "이 환자 치료 방향 제안해줘"

오케스트레이터가 실행하는 것:

[병렬 실행]
┌─ EMR 에이전트: 최근 6개월 기록 조회
├─ 검사 에이전트: 최신 Lab 결과 수집
└─ 약물 에이전트: 현재 복용 약물 목록 조회

[순차 실행]
↓ 수집된 데이터 통합
↓ 가이드라인 검색 (RAG)
↓ 감별진단 AI: 후보 진단 3개 생성
↓ 상호작용 확인: 기존 약물과 새 치료 충돌 검사
↓ 최종 요약 AI: 의사에게 전달할 보고서 생성

출력: 구조화된 치료 방향 제안 + 근거 + 주의사항
```

이 전체 흐름을 설계하고 실행하는 것이 오케스트레이션이다.

### 오케스트레이션 프레임워크

2024~2025년에 이 분야가 폭발적으로 성장했다. 주요 프레임워크:

- **LangGraph**: 그래프 기반 상태 머신으로 에이전트 흐름 설계
- **CrewAI**: 역할 기반 멀티 에이전트 프레임워크
- **AutoGen**: Microsoft가 만든 대화형 에이전트 조율
- **Prefect / Airflow**: 데이터 파이프라인 오케스트레이션을 AI에 적용
- **Claude Agents SDK**: Anthropic의 에이전트 오케스트레이션 SDK

각 프레임워크마다 강점이 다르다. 다음 글들에서 하나씩 파고든다.

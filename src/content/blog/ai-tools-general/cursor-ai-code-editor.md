토요일 오후. 전공의 민준은 지도 교수에게 레지던시 연구 데이터 분석을 맡았다. 엑셀 파일 두 개, 환자 300명의 혈압·약물 기록이 있었다. 교수가 원하는 건 그룹별 평균 비교와 그래프였다.

SPSS는 학부 때 배웠지만 라이선스가 만료돼 있었다. Python을 쓰면 된다고 들었지만, 파이썬은 "print('hello')"도 실행해본 적 없었다.

검색하다 Cursor를 발견했다. "AI가 코드를 대신 써준다"는 문구가 있었다.

## Cursor가 뭔가

Cursor는 AI가 내장된 코드 편집기다. VS Code 기반이라 인터페이스는 비슷하지만, 대화창에 원하는 것을 말하면 코드를 직접 작성해준다.

민준이 입력한 첫 문장은 이랬다.

> "엑셀 파일 두 개를 불러와서 blood_pressure_systolic 컬럼을 기준으로 그룹A와 그룹B의 평균을 비교하는 Python 코드를 써줘."

10초 안에 30줄짜리 코드가 생성됐다. pandas로 파일을 읽고, 그룹을 나누고, 평균을 계산하고, matplotlib으로 박스플롯을 그리는 코드였다. 민준은 파이썬을 몰랐지만 코드가 뭘 하는지 주석으로 설명이 달려 있었다.

실행하니 에러가 났다. 파일 경로가 달랐다. 에러 메시지를 복사해서 다시 대화창에 붙였다.

> "이 에러가 났어: FileNotFoundError: [Errno 2] No such file or directory"

Cursor가 경로 수정 방법을 알려줬다. 수정했더니 그래프가 나왔다.

## 코드를 이해하지 못한 채 완성하면

민준은 그래프를 완성했다. 하지만 한 가지 찜찜함이 있었다. 코드가 무엇을 하는지 완전히 이해하지 못한 채 결과물이 나왔다.

다음날 교수가 물었다. "t-test 가정 검토는 했어?" 민준은 코드에 t-test가 있는지도 몰랐다. 있었다. 등분산 검정 없이 그냥 돌아간 상태였다.

Cursor는 코드를 잘 써준다. 하지만 어떤 통계 방법이 이 데이터에 적절한지는 알려주지 않는다. 도구의 작동 원리와 분석 방법의 타당성은 사용자가 알아야 할 몫이다.

## Cursor가 실제로 유용한 순간

반복 작업 자동화에서 강점이 두드러진다. 같은 형식의 파일을 100개 처리하거나, 엑셀에서 특정 조건에 맞는 데이터를 추출하거나, 결과를 특정 형식으로 내보내는 작업.

민준은 이후 Cursor를 이렇게 썼다.
- 원하는 결과물의 형식을 명확하게 적어 요청
- 생성된 코드의 각 줄을 Cursor에게 설명 요청 ("이 줄이 뭘 하는 건지 설명해줘")
- 통계 가정은 교수나 통계 전공자에게 별도 확인

코드를 모르는 상태로 코드를 완성하는 경험은 강력하지만, 결과물을 그대로 쓰기 전에 내용을 검토하는 사람이 반드시 있어야 한다.

## 의료 데이터를 쓸 때 확인해야 할 것

환자 데이터를 Cursor에 입력하기 전에 한 가지를 확인해야 한다. Cursor의 AI 기능은 입력한 코드와 데이터를 서버로 전송할 수 있다. 환자 개인정보(PHI)가 포함된 데이터는 Cursor의 Privacy Mode 설정을 확인하고, 필요하다면 데이터를 완전히 익명화한 뒤 사용해야 한다.

## 끝맺음 — 두 번째 분석은 달랐다

두 번째 데이터 분석 때, 민준은 Cursor에게 코드를 받은 직후 한 줄씩 설명을 요청했다. 시간이 더 걸렸지만, 교수 앞에서 "이 분석이 왜 이 방법을 썼는지"를 설명할 수 있게 됐다. 그게 첫 번째와 달랐다.

도구가 코드를 써줄수록, 코드를 읽는 능력이 더 중요해진다.

*Privacy Mode 설정은 Cursor 공식 문서(cursor.com/privacy)를 기준으로 확인한다.*

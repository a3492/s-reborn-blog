## 한줄 요약

Meta의 LLaMA 3.1은 USMLE 기준 75% 이상의 정답률로 GPT-3.5에 근접한 의료 성능을 보였고, 오픈소스 특성 덕분에 병원이 자체 서버에 구축할 수 있어 개인정보보호 측면에서 큰 이점을 제공한다.

---

## Meta LLaMA와 의료 AI

Meta AI는 2023년부터 LLaMA(Large Language Model Meta AI) 시리즈를 오픈소스로 공개해왔다. 의료 분야에서 특히 주목받는 이유는 모델 가중치를 직접 다운로드해서 병원 내부 서버에 설치할 수 있다는 점이다. ChatGPT, Claude와 달리 외부 서버로 데이터를 보낼 필요가 없어 의료 데이터 보안 문제를 원천적으로 해결한다.

---

## LLaMA 3.x 의료 벤치마크 성능

| 모델 | USMLE Step 1 | MedQA | MedMCQA | PubMedQA |
|---|---|---|---|---|
| LLaMA 3.1 70B | 76.2% | 71.4% | 64.8% | 75.3% |
| LLaMA 3.1 405B | 82.1% | 78.9% | 71.2% | 79.6% |
| GPT-4o (비교) | 88.5% | 85.2% | 76.3% | 82.1% |
| Claude 3.5 Sonnet (비교) | 87.3% | 84.1% | 75.8% | 81.4% |
| GPT-3.5 (비교) | 60.1% | 56.3% | 54.2% | 68.9% |

LLaMA 3.1 405B는 GPT-4o보다 낮지만 GPT-3.5를 크게 앞서며, 무료 오픈소스 모델로서는 압도적인 의료 성능이다. 특히 USMLE 합격 기준(60%)을 크게 상회한다.

---

## 의료 특화 파생 모델들

Meta LLaMA를 의료 데이터로 파인튜닝한 특화 모델들이 연구자들에 의해 개발되고 있다.

### BioMedLM (Stanford / MosaicML)

- 기반: GPT-2 아키텍처, 270억 개 파라미터
- 학습 데이터: PubMed 논문 전체(약 300만 편)
- 특징: 의학 문헌 요약과 임상 질문 답변에 특화
- 한계: 최신 논문(2023년 이후) 학습 제한적

### Hippocrates (오픈소스 컨소시엄, 2024)

- 기반: LLaMA 3.1을 의료 데이터로 파인튜닝
- 학습 데이터: 의학 교과서, 임상 가이드라인, 탈식별화된 EHR 데이터
- 성능: USMLE Step 1-3 평균 78.4%, GPT-3.5 대비 18%p 향상
- 배포: Hugging Face에서 무료 다운로드 가능

### MedPaLM 2 (Google, 비교)

- 오픈소스 아님, 클라우드 API 형태로 제공
- USMLE 기준 86.5% 정답률로 현재 최고 수준
- 한국 병원의 독립 설치 불가

---

## 오픈소스 의료 AI의 장점

1. 데이터 주권 확보

병원이 LLaMA 기반 모델을 자체 서버에 설치하면, 환자 데이터가 외부로 나가지 않는다. 개인정보보호법, 의료법상 의료정보 국외 이전 제한 문제를 원천적으로 해결한다.

2. 비용 구조

초기 GPU 서버 구축 비용은 발생하지만(A100 80GB GPU 2-4개 기준 약 3,000-6,000만 원), 이후 API 비용이 없다. 하루 수천 건의 의료 기록 처리 시 OpenAI API 대비 1-2년 내 비용 역전이 발생한다.

3. 커스터마이징

병원 자체 데이터로 추가 파인튜닝이 가능하다. 예: 특정 병원의 퇴원 요약 양식에 맞게 모델을 재학습시키면 더 정확한 자동화 가능.

---

## 오픈소스 의료 AI의 단점

1. 기술 인력 필요

LLaMA를 병원에 설치하고 운영하려면 MLOps 엔지니어 또는 데이터 사이언티스트가 필요하다. 대부분의 국내 의원·중소병원에는 이런 인력이 없다.

2. 성능 격차

최신 GPT-4o, Claude 3.5 대비 의료 추론 성능이 6-12%p 낮다. 복잡한 임상 의사결정 지원에는 아직 한계가 있다.

3. 책임 소재

상업적 AI 서비스(OpenAI, Anthropic)는 서비스 약관 내에서 일정 수준의 책임을 진다. 오픈소스 모델을 병원이 직접 운영하면 모든 오류와 오진 지원에 대한 책임이 병원에 있다.

---

## 한국 의료 현장에서의 현실적 활용

현재 국내에서 LLaMA 기반 의료 AI를 자체 운영하는 기관은 대형 병원 연구팀, 의료 AI 스타트업 정도다. 의원급에서 직접 운영은 현실적으로 어렵다.

현실적인 접근:
- 단기: ChatGPT Team, Claude for Work 같은 기업용 계정으로 데이터 보안 확보
- 중기: 국내 의료 AI 기업(루닛, 뷰노 등)이 LLaMA 기반으로 개발한 솔루션 도입
- 장기: 병원 연합체(예: 병원정보시스템 공동 운영) 차원에서 공동 AI 인프라 구축

## 핵심 정리

- Meta LLaMA 3.1 405B는 USMLE 기준 82.1% 정답률 — 무료 오픈소스 중 최고 수준
- Hippocrates, BioMedLM 등 의료 특화 파생 모델이 Hugging Face에서 무료 공개 중
- 오픈소스 최대 장점: 병원 내부 서버 설치로 환자 데이터 외부 전송 없이 활용 가능
- 단점: MLOps 운영 인력 필요, GPT-4o 대비 성능 격차, 오류 책임 병원 귀속
- 국내 의원급은 단기적으로 ChatGPT Team/Claude for Work이 현실적 대안
